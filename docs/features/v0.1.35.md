## Batch Intent

`KodaX owns runtime behavior; Space owns visibility, review, recovery, and user control.`

v0.1.35 turns three KodaX intelligence primitives into desktop workflows: effort-first provider routing (instead of a provider/model matrix), a verifiable advisor consult attached to risky review surfaces, and a skill self-improvement review queue so agent behavior never changes silently.

---

## FEATURE_091: Effort-First Provider Routing UI

### 需求概述

Provider/model selection is becoming too complex for a matrix. KodaX F233 makes reasoning effort the first-class control; Space should expose that as a task-aware UI, including a clearer autonomy ladder around the existing `plan` / `accept-edits` / `auto` modes.

### 影响范围

- Renderer provider/model/effort controls (replaces matrix-heavy selection).
- Session-level override state.
- Autonomy-mode copy across the existing mode selector.

### 技术方案

Replace matrix-heavy controls with:

- task intent selector or inferred task class,
- effort levels tied to capability metadata,
- provider/model badges,
- "why this route" explanation,
- per-session override,
- fallback when selected provider lacks a capability.
- a clearer autonomy ladder around the existing `plan` / `accept-edits` / `auto` modes, including concise risk copy and the reason a mode may still ask for approval.

### 接口契约

- Consumes KodaX F233 effort-first routing + capability metadata.

### 实现步骤

_待 Plan 阶段细化（depends on KodaX F233 capability/effort metadata）._

### 验收标准

- Users can choose low/standard/high effort without knowing provider-specific flags.
- Space explains selected provider/model/effort in one compact line.
- Unsupported effort/model combinations are impossible or clearly disabled.
- Users can understand the current autonomy level without opening advanced settings or reading slash-command help.

## FEATURE_092: Advisor Consult Review Surface

### 需求概述

KodaX will expose a verifiable advisor consult primitive. Space can make it useful by turning it into a review action for risky work, with results attached to the reviewed object rather than dumped into normal chat.

### 影响范围

- "Ask Advisor" actions added to high-risk review surfaces (plan, diff, workflow draft, artifact, release notes/PR).
- Advisor result attachment model.

### 技术方案

Add "Ask Advisor" actions for:

- plan review,
- diff review,
- workflow draft review,
- artifact/source faithfulness review,
- release note or PR description review.

Advisor output should include:

- verdict,
- evidence references,
- disagreement points,
- model/provider used,
- confidence and limitations.

### 接口契约

- Consumes the KodaX verifiable advisor-consult primitive (cross-model verification).

### 实现步骤

_待 Plan 阶段细化（depends on the KodaX advisor-consult primitive）._

### 验收标准

- Advisor consult is available from high-risk review surfaces.
- Advisor results are attached to the reviewed object, not dumped into normal chat.
- Users can accept, dismiss, or ask the main session to revise based on advisor feedback.

## FEATURE_093: Skill Self-Improvement Review Surface

### 需求概述

KodaX self-improvement can propose skill changes. A desktop app needs a review queue so users understand how agent behavior is changing, and no proposed skill should silently change active behavior.

### 影响范围

- New Skill Changes panel in the renderer.
- Builds on the v0.1.26 `/learn` / `/skill pending` learning-store surface.

### 技术方案

Add a Skill Changes panel:

- proposed skill edits,
- diff view,
- generated rationale,
- test/eval result,
- activate, reject, edit, rollback,
- behavior impact summary.

No proposed skill should silently change active behavior unless the user has enabled an explicit policy.

### 接口契约

- Consumes the KodaX skill self-improvement proposal store, diff, and test/eval results (extends the procedural-learning store used by `/learn`).

### 实现步骤

_待 Plan 阶段细化._

### 验收标准

- Skill changes are reviewable before activation.
- Activation and rollback are explicit.
- Test/eval output is visible when available.
- Active skill version is inspectable.
