## FEATURE_076: UI Language Switch and i18n Foundation

## 背景

客户反馈 KodaX-Space 没有中文/英文显示切换。这个反馈成立，而且不应该只用一个「中文 / English」按钮解决。当前 app 已经存在大量硬编码 UI 文案：全局菜单、Settings、左右侧栏、popout 面板、权限弹窗、toast、命令面板、HelpOverlay、workflow 面板等都直接写死英文或中英混排。越晚处理，后续回填成本越高。

本 feature 的核心结论：

**做一个全局 i18n 基础设施，并把语言切换作为用户偏好接入；第一版覆盖全局菜单和高频 UI 面板，不改变用户内容、模型输出和工具输出。**

2026-06-22 更新：为了让 v0.1.20 先具备可用的中/英切换，F104 从本文拆出一个 Display Language MVP。本文后续仍作为完整 i18n foundation 的设计源，重点从「先做第一个可见切换」调整为「把 F104 的基础设施扩成全量 typed locale 和质量门禁」。

## 产品决策

F076 只提供 3 个用户可见语言模式：

| Mode     | 含义                 | effective locale                                                      |
| -------- | -------------------- | --------------------------------------------------------------------- |
| `system` | 跟随系统首选 UI 语言 | 系统首选语言含简体中文或 generic Chinese 时为 `zh-CN`，否则为 `en-US` |
| `zh-CN`  | 强制简体中文 UI      | `zh-CN`                                                               |
| `en-US`  | 强制英文 UI          | `en-US`                                                               |

默认值：`system`。

入口：`Settings > Preferences > Language`。不在标题栏、侧栏、底栏常驻一个语言切换按钮。命令面板可以加 `Preferences: Change Language`，但它只打开 Preferences，不做第二套切换入口。

切换行为：

- renderer UI 立即更新；
- main 进程重建 Electron native menu；
- app-owned menu label 必须跟随语言；
- Electron/OS role 自带 label 如果平台不支持即时刷新，可以提示「重启后完整生效」，但这只能是兜底。

重要边界：

- UI language 不等于 assistant response language；
- 不自动改变模型回复语言；
- 不翻译用户消息、assistant 消息、tool stdout/stderr、文件路径、代码、Markdown 正文、model/provider/tool 名称；
- 原始 SDK 错误可以用本地化外壳包一层，但 raw message 本身不强行翻译。

## 目标

1. 建立 main 和 renderer 共用的 typed i18n 基础设施。
2. 增加 `system` / `zh-CN` / `en-US` 语言偏好。
3. 从 stored preference + OS preferred languages 解析 effective locale。
4. 覆盖全局菜单、Settings、左右侧栏、popout 面板、弹窗、toast、命令面板、HelpOverlay、workflow 面板等高频 UI。
5. 把语言偏好持久化到 `~/.kodax/space/settings.json`，并迁移现有 settings schema。
6. 增加测试和 CI guard，避免新 UI 文案绕过 locale。
7. MVP 零运行时新依赖，保持现有 Electron + React + Zustand 架构。

## 非目标

- 不做机器翻译。
- 不做云端 language pack marketplace。
- 不做繁体中文、日文、韩文、企业术语表。
- 不翻译 LLM 回复、用户内容、tool 输出、文件内容、artifact 正文、代码块、文件路径、model/provider/tool 名。
- 不在 MVP 引入完整 ICU/Fluent parser。
- 不做常驻 app chrome 语言切换器。

## 用户故事

1. 中文系统用户首次打开 Space，菜单、设置、面板标题、按钮、空状态默认显示简体中文。
2. 英文系统上的中文用户可以在 Preferences 改成简体中文，UI 立即生效，重启后仍保持。
3. 中文系统上的英文用户可以强制 English，重启后仍保持。
4. `system` 模式下，用户改变 OS 首选语言并重启 Space，Space 会按新系统语言解析 UI 语言。
5. 开发者新增 UI 文案时有 TypeScript key autocomplete；某个 locale 缺 key 时测试失败。

## 架构设计

### Shared i18n package

新增 workspace package：

```text
packages/space-i18n/
  package.json
  tsconfig.json
  src/
    index.ts
    types.ts
    resolve.ts
    translate.ts
    format.ts
    locales/
      en-US.ts
      zh-CN.ts
```

公共 API：

```ts
export type SupportedLocale = 'en-US' | 'zh-CN';
export type LanguageMode = 'system' | SupportedLocale;
export type MessageId = keyof typeof enUS;

export const SUPPORTED_LOCALES: readonly SupportedLocale[];
export const DEFAULT_LOCALE: SupportedLocale;

export function resolveLocale(input: {
  mode: LanguageMode;
  preferredLanguages: readonly string[];
}): SupportedLocale;

export function createTranslator(locale: SupportedLocale): {
  locale: SupportedLocale;
  t(id: MessageId, vars?: Record<string, string | number>): string;
  formatDate(value: Date | number, options?: Intl.DateTimeFormatOptions): string;
  formatNumber(value: number, options?: Intl.NumberFormatOptions): string;
};
```

字典示例：

```ts
export const enUS = {
  'settings.title': 'Settings',
  'settings.subtitle': 'KodaX Space controls',
  'settings.language.title': 'Language',
  'settings.language.description': 'Menus, panels, dialogs, and app controls.',
  'settings.language.system': 'Follow system',
  'settings.language.zhCN': '简体中文',
  'settings.language.enUS': 'English',
} as const;
```

变量文案优先使用完整 message 或 function：

```ts
'settings.language.systemEffective': ({ localeName }) => `Current: ${localeName}`,
```

不要拼碎片：

```ts
t('current') + ': ' + localeName;
```

### Locale resolution

解析规则：

1. `languageMode === 'zh-CN'` 时直接用 `zh-CN`。
2. `languageMode === 'en-US'` 时直接用 `en-US`。
3. `languageMode === 'system'` 时按 OS preferred languages 顺序解析。
4. 下列值映射到 `zh-CN`：
   - `zh-CN`
   - `zh-Hans`
   - `zh-Hans-*`
   - generic `zh`
5. 其它语言映射到 `en-US`。
6. `C`、`POSIX`、空字符串、未知值映射到 `en-US`。

说明：MVP 不把 `zh-Hant` 静默映射到简体中文。繁体中文是后续明确支持项，不应让繁中用户误以为已经获得完整本地化。

### Settings persistence

当前 settings：

```ts
{
  version: 1,
  defaultWorkspace: string
}
```

F076 迁移到：

```ts
{
  version: 2,
  defaultWorkspace: string,
  languageMode: 'system' | 'zh-CN' | 'en-US'
}
```

迁移规则：

- settings 文件不存在：默认 `languageMode: 'system'`。
- v1 文件：保留 `defaultWorkspace`，补 `languageMode: 'system'`，下一次写入时落成 v2。
- 非法 languageMode：fallback 到 `system`，写 warn，不阻塞启动。

IPC 更新：

```ts
settings.get -> {
  defaultWorkspace: string;
  languageMode: LanguageMode;
  effectiveLocale: SupportedLocale;
  preferredSystemLanguages: string[];
}

settings.setLanguageMode -> {
  languageMode: LanguageMode;
} -> {
  languageMode: LanguageMode;
  effectiveLocale: SupportedLocale;
}
```

`settings.setDefaultWorkspace` 保留，但返回值也升级为完整 settings payload。

### Main process

main 进程负责 persisted language preference 和 native menu。

新增：

```text
apps/desktop/electron/i18n/locale.ts
apps/desktop/electron/i18n/menu.ts
```

职责：

- `settingsStore.load()` 读取 languageMode；
- `app.getPreferredSystemLanguages()` 获取 OS preferred languages；
- 用 `resolveLocale()` 得到 effective locale；
- 创建 `createTranslator(effectiveLocale)`；
- 把 `main.ts` 里的 menu template 抽成 `buildApplicationMenu(t)`；
- 启动时设置 native menu；
- `settings.setLanguageMode` 成功后重建 native menu；
- 推送 settings changed event 给 renderer。

伪流程：

```ts
const settings = await settingsStore.load();
const effectiveLocale = resolveLocale({
  mode: settings.languageMode,
  preferredLanguages: app.getPreferredSystemLanguages(),
});
Menu.setApplicationMenu(buildApplicationMenu(createTranslator(effectiveLocale)));
```

### Renderer

新增：

```text
apps/desktop/renderer/src/i18n/I18nProvider.tsx
apps/desktop/renderer/src/i18n/useI18n.ts
apps/desktop/renderer/src/i18n/bootstrap.ts
```

启动流程：

1. 读取 `localStorage['kodax-space.languageMode']` 作为 first-paint cache；
2. 用 `navigator.languages` 本地解析 initial locale；
3. 调 `settings.get` 获取 main 权威状态；
4. 若 main 返回不同 languageMode/effectiveLocale，更新 provider；
5. 用户在 Preferences 切换语言时，调用 `settings.setLanguageMode`，成功后更新 provider 并写 localStorage cache。

`App.tsx`：

```tsx
<I18nProvider initialLocale={bootstrapLocale}>
  <AppShell />
</I18nProvider>
```

组件用法：

```tsx
const { t } = useI18n();
return <h2>{t('settings.title')}</h2>;
```

非 React 用法：

- command source builder 接收 `t`，不要在 module top-level 固化 translated label；
- popout/action registry 中的 label 需要在 render 时计算；
- dynamic action 的 `searchText` 同样本地化，保证命令面板中英都可搜。

### Message key 规范

使用语义 key，不用英文句子作为 key：

```text
settings.title
settings.preferences.tab
settings.providers.tab
sidebar.projects.title
session.menu.rename
permission.allowOnce
workflow.policy.autoStart.confirm
```

规则：

- key 是稳定契约；
- 删除 key 前必须确认所有 locale 和调用方；
- provider/model/tool/file path/env var 不进入 locale；
- dynamic error 先映射 stable error code；
- unknown error 用本地化外壳 + sanitized raw message。

### Formatting

使用 effective locale 的 `Intl` 处理：

- 日期和时间；
- 相对时间；
- 数字和百分比；
- 文件大小，如果已有 helper 则改造成 locale-aware。

保持原样：

- 文件路径；
- 命令名；
- model/provider/tool 名；
- env var；
- code / markdown / artifact body。

## 风险与缓解

| 风险                            | 缓解                                                                                                   |
| ------------------------------- | ------------------------------------------------------------------------------------------------------ |
| 半迁移导致 UI 中英混杂          | 先迁顶层 chrome 和高频路径；保留覆盖清单；新高频字符串 review 时必须走 locale。                        |
| 启动时语言闪烁                  | renderer 用 localStorage + `navigator.languages` 做 first-paint cache，再与 main `settings.get` 对齐。 |
| native menu role 被破坏         | role item 不重写行为，只翻译 app-owned label；macOS 单测/手测 copy paste select all。                  |
| 拼接句子导致中文别扭            | 使用完整句子 key 或 message function；review rule 禁止拼碎片。                                         |
| settings migration 影响用户配置 | zod union parse v1/v2 + atomic write；malformed file 单测。                                            |
| 过度翻译 raw technical content  | 明确保留 provider/model/tool/path/code/raw output 原样。                                               |

## 开放问题

1. `zh-Hant` 是否 fallback 到 English 还是 Simplified Chinese？建议：MVP fallback English，直到明确支持繁中。
2. Command Palette 是否加 `Preferences: Change Language`？建议：加，但只打开 Settings。
3. assistant response language 是否跟随 UI？建议：不在 F076 做，后续做独立 preference。
4. renderer localStorage 是否能作为 main 不可用时的真实来源？建议：不能；main settings 是 source of truth，localStorage 只做 first-paint cache。

## 后续 feature

### F077：dev-only pseudo-locale / 文案扩展 QA

增加只在 dev/test 可用的 pseudo-locale，用拉长字符串、加边界标记和模拟宽字符的方式暴露 UI 溢出、按钮挤压和布局抖动。该 locale 不进入用户设置，不参与正式语言列表。

验收：开发者可以通过 env/测试 flag 开启 pseudo-locale；核心 UI 在 960x600 和 1280x800 下没有明显重叠；截图回归可覆盖设置、侧栏、权限弹窗、workflow 面板。

### F078：i18n scanner 从 advisory 升级为 CI required

把 F076 中的硬编码文案扫描从提醒型脚本升级为 CI gate。允许 raw technical content、model/provider/tool/path/code 保持原文，但高频用户可见 UI 文案必须走 typed locale key。

验收：新增 JSX/TSX 用户可见文案绕过 locale 时 CI 失败；允许名单有注释和 owner；locale key parity test 与 scanner 一起运行。

### F079：繁体中文支持

在 `zh-CN` / `en-US` 稳定后增加 `zh-Hant`。不使用简体中文静默 fallback 伪装完整支持；繁中必须有独立字典、fallback、排版 QA 和目标系统语言解析。

验收：`system` 模式下繁中系统可解析到 `zh-Hant`；三套 locale key 完全一致；繁中 UI 主路径无明显截断或错字。

### F080：Assistant response language preference

增加模型回答语言偏好，并明确与 UI display language 分离。用户可以让 UI 用英文、assistant 默认中文，或反过来。该设置只影响 Space 注入的会话偏好，不翻译用户内容、tool 输出或 artifact 正文。

验收：设置项独立于 UI language；新 session 能带入回答语言偏好；已有 session 不被静默改写；Quick Ask 与 Coder/Partner 的默认策略一致且可覆盖。
