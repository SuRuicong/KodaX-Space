# v0.1.6 — Persistence follow-up

> 与 KodaX SDK ≥ 0.7.42 同步发布。本版本只有一个 feature：把 F033 (Sessions tree + fork + rewind) 从 in-memory 升级到磁盘持久化，接 KodaX SDK 在 0.7.42 出的公开 sessions API。
>
> 触发条件：KodaX SDK 发布 0.7.42 (或更高) 含 listSessions/loadSession/forkSession/rewindSession 公开 API。

## 版本目标

F033 在 v0.1.1 alpha.1 期间作为 **in-memory only** 落地——重启 desktop session 会丢，且无法看到/操作 KodaX REPL 已创建的 session。v0.1.6 把数据源换成 SDK 提供的持久化 API，让 desktop 与 REPL 真正共享同一份 session 历史。

---

## FEATURE_038: F033 Sessions 持久化升级（接 KodaX SDK ≥ 0.7.42）

### 需求概述

让 desktop 的 sessions tree / fork / rewind 走 KodaX SDK 公开持久化 API，与 REPL 共享 `~/.kodax/sessions/`。F033 已经搭好的 UI / IPC / appStore 层不动，只把 `host.fork()` / `host.rewind()` / `host.list()` 的内部实现从 in-memory Map 换成 SDK 调用。

### 影响范围

- `apps/desktop/electron/kodax/host.ts` — fork/rewind/list 内部实现替换
- `apps/desktop/electron/kodax/session-store.ts` (新建) — 包装 SDK `listSessions/loadSession/forkSession/rewindSession`，加 desktop 端 cache + 文件监听
- `apps/desktop/electron/ipc/session.ts` — session.list 返回 SDK + in-memory 合并视图（in-flight session 仍来自 in-memory map）
- `apps/desktop/renderer/src/store/appStore.ts` — fork buffer 复制改为 "从 SDK loadSession 拿 turns 重放成 events" 而非直接复制 renderer 内存
- `package.json` — 升 `@kodax-ai/kodax` 到 ≥ 0.7.42

### 前置依赖

- KodaX SDK 0.7.42 发布，含四个公开操作：`listSessions({projectRoot?, includeArchived?})` / `loadSession(id)` / `forkSession(srcId, forkPointTurnIdx)` / `rewindSession(id, rewindPastTurnIdx)`（可选 `watchSessions()` 文件监听）
- KodaX SDK 团队确认 on-disk schema 稳定（schema-version 字段或显式契约）

### 技术方案

**核心切换点（host.ts）**：

| 方法 | 当前 (in-memory) | 升级后 |
|------|------------------|--------|
| `kodaxHost.list()` | 仅返回 in-memory Map | 合并 `sdk.listSessions({projectRoot})` + in-memory in-flight；in-flight 优先（更"新"） |
| `kodaxHost.fork(srcId, idx)` | 复制 in-memory metadata 到新 sessionId | `sdk.forkSession(srcId, idx)` → SDK 返回新 sessionId；host 加载新 session 入 in-memory Map 作为活动 session |
| `kodaxHost.rewind(id, idx)` | cancel in-flight + renderer 截 buffer | `sdk.rewindSession(id, idx)` (await + 处理 session_busy/invalid_index/not_found) |

**Renderer fork buffer 重建**：当前实现是 `forkSessionBuffers` 直接复制 renderer 内存 events。升级后改成：
1. fork IPC ack 拿 newSessionId
2. 调 `sdk.loadSession(newSessionId)` 拿 turns[]
3. 把 turns[] 重放成 SessionEvent 序列填进 `eventsBySession[newSessionId]`

这样 desktop 重启后 fork child 仍能 reload，不依赖 renderer in-memory 状态。

**文件监听**：用 SDK `watchSessions()` (如果 SDK 出) 或 5s 轮询，在 session 文件变更时刷新 sidebar——让用户在 REPL 里跑的 session 也能在 desktop sidebar 看到。

### 接口契约（依赖 SDK 0.7.42 ready）

```ts
import {
  listSessions,
  loadSession,
  forkSession,
  rewindSession,
  watchSessions,  // 可选
  type KodaXPersistedSession,
} from '@kodax-ai/kodax';
```

具体类型形态由 KodaX SDK 团队最终确定；本 feature 实施时按 SDK 实际 `.d.ts` 接入。

### 迁移路径

- v0.1.1 已发布带 F033 in-memory 的 alpha 用户：升 v0.1.6 后看到的是 KodaX REPL 累积的 session 历史；他们 v0.1.1 期间 fork 出的 in-memory session（重启后已丢失）自然 0 影响
- 无 schema 破坏：sessionMetaSchema 的 `parentSessionId`/`forkPointTurnIdx` 字段都是 optional，SDK 注入仍兼容
- IPC schema 不动（sessionForkChannel / sessionRewindChannel 字段已经按 SDK 形态预留）

### 验收标准

- [ ] desktop 启动后，sidebar 显示 KodaX REPL 已创建的 sessions（按 projectRoot 过滤后）
- [ ] 在 REPL 里 fork 一个 session，desktop sidebar 自动出现新 child（≤ 5s 内）
- [ ] 在 desktop fork → REPL 端 listSessions 也能看到新 session
- [ ] rewind 后 desktop + REPL 看到的 turns 数量一致
- [ ] desktop 重启后所有 session（含 fork child）仍在
- [ ] SDK rewind 返回 'session_busy' 时 desktop UI 给出明确提示，不静默成功
- [ ] 文件损坏的 session 跳过不崩，sidebar 显示其他正常 session

### 实现步骤

1. 升 `@kodax-ai/kodax` 到 ≥ 0.7.42（package.json + npm install）
2. 写 `apps/desktop/electron/kodax/session-store.ts` 包装 SDK，加 LRU cache + 文件监听抽象
3. 重写 `kodaxHost.list/fork/rewind` 内部实现，外部 API 形态不变
4. 重写 `appStore.forkSessionBuffers`：从 `sdk.loadSession` turns 重放成 events
5. 加测试：mock SDK 返回值 → 验证 host/IPC/appStore 行为
6. e2e 验证：desktop 与 REPL 共享 session 文件，互相能看到对方的 fork
7. 文档：[docs/features/v0.1.1.md#033] 加一句"已在 v0.1.6 升级为持久化版"

### 风险

- **SDK API shape 变更**：v0.1.1 in-memory 实现已经按 `KodaXPersistedSession` 类型设计 hook 点，但 SDK 实际出来若与 [SDK-REQUEST 文档](../external/KODAX-SDK-REQUEST-session-persistence.md) 形态不一致，可能需要在 session-store 层适配
- **REPL 并发写**：rewindSession 应当处理 REPL 正在跑同 session 的情况（返回 session_busy）。SDK 层若未做并发锁，desktop 这边只能加竞态防御
- **大 session 加载延迟**：`loadSession` 拿 turns[] 可能很大（数十 MB）。要么 SDK 分页加载，要么 desktop 在 fork 后 lazy 加载（先显示 metadata，事件流懒加载）

### 不做的事

- 不在本版本做 session 全文搜索（留 v0.2.x）
- 不在本版本做 archived session 单独面板（F033 设计文档里提过——仍按 sidebar 分组实现，不做独立 popout）
- 不重构 ManagedSession in-memory 层——in-flight 仍走 host Map，只有"历史 + fork 元数据 + truncate"走 SDK
