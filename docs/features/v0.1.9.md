## 关键技术决策

### OC-31 image paste 走 KodaX SDK 原生 inputArtifacts

SDK 0.7.45 已经暴露 `KodaXContextOptions.inputArtifacts: KodaXInputArtifact[]`,实测 dist 里 `buildPromptMessageContent(prompt, artifacts)` 真把 image 转 `{type:'image', path, mediaType}` multimodal content block。Space 不需要拼协议 / 不需要 base64 inline,只需 main 落盘 → 把 path 塞进 artifacts → SDK 自己读文件喂给 provider transport。

落盘策略: `app.getPath('temp')/kodax-space/clipboard/<sid>/<ts>.png`,dir mode 0o700 / file mode 0o600,session.delete + disposeAll best-effort 清盘。`session.send` 调用前对 artifacts[].path 做 sandbox 校验(必须在 `<root>/<本次 sid>/` 之内),防恶意 renderer 传 `/etc/passwd`。

### KX-I-02 Director: 不抢 + per-(session,kind) 一次

director 是 React useEffect 听 `eventsBySession[currentSid]`,跑 pure function `decideAutoPromote(events, activePopout, promoted) → kind | null`。规则:

- activePopout 非 null → 不抢
- 已 promoted 过该 kind → 不重复
- 优先级 tasks > plan > diff (一帧多触发取一个)

用户**手动**开 popout (CommandToolbar / ⤢ / requestedPopout 三条路径) 也 mark promoted,避免下一秒 director 再"自动"开同一个被用户刚关掉的 popout。

### F040 项目拖排: 保留 "current 排首"

`projectOrder.length === 0` 时走旧逻辑 (lastUsedAt 倒序 + current 排首);非空时按 projectOrder 排,但**仍然 pin current 到最顶**。这跟 IDE/Codex workspace switch 体感对齐 — 切到老项目时焦点不应该跳到中段。

### SDK 0.7.46 适配 — 撤掉两轮 workaround

v0.1.9 早期 (0.7.45 时):
- 加 `includeArchived: true` 强制 slow path 绕 fast-path bug

v0.1.9 中期 (0.7.46 第一版认知):
- 改用 `before: '2999-...'` sentinel 触发 slow path

v0.1.9 最终 (0.7.46 真正 fix 之后):
- 撤掉所有 workaround,纯净调用 listSessions
- SDK 端 storage.list 加 `this.hostCwd ?` 守门:caller 不传 cwd → currentGitRoot=null → 走"扫所有 projectKey 目录"分支
