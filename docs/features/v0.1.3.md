## FEATURE_019: 主题（明 / 暗 / 系统跟随）

### 需求概述

支持三种主题：明、暗、跟随系统。覆盖 Monaco / xterm / shadcn 全套组件。

### 影响范围

- 修改：`apps/desktop/renderer/theme/`（建立 token 系统）
- 修改：所有 UI 组件用 CSS variables
- 修改：Monaco theme 绑定、xterm theme 绑定
- 新建：设置页主题切换 UI

### 技术方案

- Tailwind `dark:` variant + CSS variables
- shadcn/ui 自带 dark mode 支持
- Monaco：`vs-dark` / `vs` 主题随之切
- xterm：自定义 ITheme 跟随 CSS variables
- 用户偏好写 `~/.kodax/space/preferences.json`：`theme: 'light' | 'dark' | 'system'`
- main 监听 `nativeTheme.shouldUseDarkColors` 变化通知 renderer

### 接口契约

```typescript
'preferences.setTheme' → { theme: 'light' | 'dark' | 'system' } → { ok }
'preferences.get' → { preferences: SpacePreferences }

// 事件
{ kind: 'theme_changed', resolved: 'light' | 'dark' }
```

### 实现步骤

1. 建立 design token（color / spacing / radius）
2. 改造现有组件用 CSS variables
3. Monaco / xterm 主题绑定
4. 设置页 toggle
5. nativeTheme 监听
6. 启动时根据 preferences 应用

### 验收标准

- [ ] 三种主题切换覆盖 100% 组件（无残留浅色 hardcode）
- [ ] 系统切换时跟随 < 200ms
- [ ] Monaco diff 在暗色下可读性 OK
- [ ] xterm 颜色不刺眼（暗色背景下）

## FEATURE_020: 桌面通知

### 需求概述

长任务完成、需要审批时发系统级通知，用户即使切到别的应用也能感知。

### 影响范围

- 新建：`apps/desktop/electron/notifications.ts`
- 修改：Permission 触发点、Task complete 事件点

### 技术方案

- Electron `new Notification({ title, body, icon })` 调用 OS 通知中心
- 触发场景：
  - Permission 请求超过 5s 用户未操作 → 通知
  - Task 完成（H1/H2 终态）→ 通知（"Session abc123 finished in 2m 30s"）
  - Session error → 通知
- 用户点通知 → focus Space + 跳到对应 session
- 静音设置：免打扰时段、特定 session 静音

### 接口契约

```typescript
'notification.preferences.get' → { notifications: NotificationPrefs }
'notification.preferences.set' → { prefs: NotificationPrefs } → { ok }

// 内部 (main use)
notify(opts: { title, body, sessionId?: string, urgency: 'normal'|'critical' })
```

### 实现步骤

1. notifications.ts 包装 + 排队 (debounce)
2. 接入 KodaX 事件流（complete、error）
3. Permission 超时检测
4. 点击通知 focus 窗口 + 跳 session
5. 设置页通知偏好（启停、免打扰、声音）

### 验收标准

- [ ] 关 Space 主窗口（最小化）任务完成时通知到位
- [ ] 点通知正确 focus 到对应 session
- [ ] 1 秒内多个通知合并为 1 条（不轰炸）
- [ ] 免打扰时段内不通知
- [ ] macOS / Win / Linux 都用各自 OS 通知中心

## FEATURE_021: `.mcpb` 一键安装

### 需求概述

支持 Anthropic 的 `.mcpb`（Desktop Extension）格式：拖拽 / 点开文件 → main 解析 manifest → KodaX 注册 → UI 列表新增。与 Claude Desktop 兼容。

### 影响范围

- 新建：`apps/desktop/electron/mcpb-installer.ts`
- 修改：MCP 管理 UI 加 "Install extension..." 按钮
- 关联：[FEATURE_013](v0.1.1.md#feature_013-mcp-管理-v1列表--启停) 的 MCP 列表

### 技术方案

- `.mcpb` 实际是 zip：`manifest.json` + server 实现文件
- manifest schema 跟 Anthropic 公开规范
- main 解压到 `~/.kodax/mcpb-extensions/<name>-<version>/`
- 调 KodaX 内核 MCP 注册 API
- 卸载：删目录 + KodaX 取消注册
- 版本管理：同名 extension 升级走 manifest 中 version 比较
- 签名验证：M2 加（v0.1.3 暂不验签）

### 接口契约

```typescript
'mcpb.install' → { filePath: string } → { extensionId, name, version }
'mcpb.uninstall' → { extensionId } → { ok }
'mcpb.list' → { extensions: McpbExtension[] }

// 拖拽 / Open With 自动触发：renderer 监听 file drop / main 监听 OS file association
```

### 实现步骤

1. mcpb-installer 解压 + manifest 校验
2. KodaX MCP 注册集成
3. UI: Install / Uninstall / Update 按钮
4. 文件关联：macOS / Win 把 `.mcpb` 默认应用设为 Space
5. 拖拽到主窗口安装
6. 安装失败友好错误（manifest 缺字段等）

### 验收标准

- [ ] 跑通至少 3 个 Anthropic 官方 `.mcpb` 包安装
- [ ] 重复安装同包升级 version
- [ ] 拖拽 + 双击文件都触发安装
- [ ] 卸载干净（无残留 disk / KodaX 注册项）

## FEATURE_022: 自动更新（Squirrel）

### 需求概述

应用自动检查更新、下载、安装新版本。用户不用手动追 release。

### 影响范围

- 新建：`apps/desktop/electron/auto-update.ts`
- 新建：发布管道（GitHub Releases + update.json manifest）
- 修改：electron-builder 配 squirrel 选项

### 技术方案

- macOS：electron-updater + Squirrel.Mac（基于 GitHub Releases 或自建 endpoint）
- Win：electron-updater + Squirrel.Windows
- Linux：v0.1.3 不做（M3 .deb 自更）
- 通道：`stable` / `beta`（用户可选）
- 流程：启动后 + 每 6h 检查 → 后台下载 → 提示用户安装（"Restart to update"）
- 不强制更新（用户可推迟）
- 更新前确认 session 都已落盘

### 接口契约

```typescript
'update.check' → void → { available: boolean, version?: string, releaseNotes?: string }
'update.downloadAndInstall' → void → { ok }
'update.setChannel' → { channel: 'stable' | 'beta' } → { ok }

// 事件
{ kind: 'update_available', version, releaseNotes }
{ kind: 'update_downloaded', version }
```

### 实现步骤

1. 配 electron-updater
2. 启动检查 + 周期检查
3. UI：版本号 + "Check for updates" + 通道切换
4. 更新前 force-flush KodaX session 状态
5. 测试：从 v0.1.2 → v0.1.3 升级流程
6. 灰度发布：v0.1.3 通道先发 beta，2 周后稳定

### 验收标准

- [ ] 启动 30s 内静默完成更新检查
- [ ] 有新版本时 toast 提示，不打断当前 session
- [ ] 用户点 "Install" 后下载进度可见
- [ ] 重启更新成功率 > 99%（macOS + Win）
- [ ] 通道切换立即生效
