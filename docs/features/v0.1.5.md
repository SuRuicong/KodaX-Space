## FEATURE_027: 代码签名 + notarize

### 需求概述

macOS：用 Apple Developer ID 签名 + 通过 notarize。Win：用 EV 代码签名证书。让用户首启不必右键 Open 或点 "Run anyway"。

### 影响范围

- 修改：`electron-builder.yml`
- 修改：CI release workflow（注入签名密钥）
- 新建：`scripts/sign-mac.mjs`、`scripts/sign-win.mjs`

### 技术方案

- macOS：
  - Apple Developer 账号（年 $99）
  - 创建 Developer ID Application 证书 + Developer ID Installer 证书
  - Hardened Runtime + Entitlements（KodaX-host 需要 keychain access、jit、allow-unsigned-executable-memory for node-pty）
  - `electron-builder` 自带 codesign 流程
  - notarize：`xcrun notarytool submit ... --wait`
- Win：
  - EV 代码签名证书（如 DigiCert / GlobalSign，年 ~$300+）
  - HSM token / cloud signing service
  - `electron-builder` 集成 SignTool
- CI：把签名凭据存到 GitHub Encrypted Secrets，仅 release workflow 可见
- 不签名的 dev build 仍保留（CI artifact for PR），但 release 必签

### 接口契约

无外部 API。产出：

- macOS: `KodaX-Space-0.1.5-universal.dmg`（notarized）
- Win: `KodaX-Space-Setup-0.1.5.exe`（EV signed）

### 实现步骤

1. Apple Developer Program 注册 + 证书创建
2. 购 EV 证书 + HSM 设置
3. entitlements.plist 配置（keychain、jit、node-pty 需要的项）
4. CI release workflow：注入证书 + 签 + notarize
5. 验证：从签后安装包装，confirm SmartScreen / Gatekeeper 无警告
6. 文档化签名流程（团队成员都能跑 release）

### 验收标准

- [ ] macOS Sonoma+ 首启无"未知开发者"警告
- [ ] Win 11 双击安装包 SmartScreen 不拦截
- [ ] `codesign --verify --strict` 通过
- [ ] `xcrun stapler validate` 通过
- [ ] Release 流程文档完整可重现

## FEATURE_028: 隐私政策 + 文档站

### 需求概述

发布前的合规与用户信任基础：

1. 隐私政策（中英）说明：数据本地存储 / API key 走 OS keychain / LLM 流量走 provider / telemetry opt-in
2. 文档站：quickstart、provider 配置指南、MCP 用法、Skill 教程、ADR 公开链接

### 影响范围

- 新建：`apps/web-docs/`（独立子项目）
- 新建：`docs-content/`（Markdown 内容）
- 新建：法务审过的 `privacy-policy.md`、`terms.md`

### 技术方案

- 文档站用 VitePress 或 Nextra（轻量、Markdown-first）
- 部署：GitHub Pages 或 Cloudflare Pages
- 内容来源：复用 `docs/PRD.md`、`docs/HLD.md`、`docs/ADR/*` 中的公开部分（剔除内部决策细节）
- 隐私政策模板：参考 Anthropic / OpenAI 公开版本，自家化
- 多语言：中英双语（Space 自身也准备 i18n）
- 文档内链接 Space app 内 `Help → Online docs` 直跳

### 接口契约

文档站 URL 结构（占位域名）：

- `kodax.space/docs/quickstart`
- `kodax.space/docs/providers`
- `kodax.space/docs/mcp`
- `kodax.space/docs/skills`
- `kodax.space/docs/repointel`
- `kodax.space/privacy`
- `kodax.space/terms`

### 实现步骤

1. 选 VitePress + Tailwind 主题
2. 写 quickstart（5 分钟跑通第一个 session）
3. 写 provider catalog（12 个 + 自定义）
4. 写 MCP 用法（含 `.mcpb`）
5. 写 Skill 教程（内置 + 自定义）
6. 写隐私政策（法务过一遍）
7. 写 terms（标准 Apache 2.0 + 商业使用条款）
8. i18n 中英
9. CI: 文档站 PR preview + 生产部署
10. Space app 内 Help 菜单链接

### 验收标准

- [ ] quickstart 跑通用户实测 < 5 分钟成功（5 个非团队用户验证）
- [ ] 隐私政策中英对齐，覆盖所有数据类型
- [ ] 12 个 provider 每个有独立配置页
- [ ] MCP / `.mcpb` 用法有示例
