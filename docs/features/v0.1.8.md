## FEATURE_043: 项目级 contextmenu

### 需求概述

对齐 codex 形态：右键项目节点弹出菜单，提供 Rename / Archive / Remove from Space 三项。
Remove 只从 KodaX Space 的 recent 列表移除，**不删除磁盘文件夹**。Archived 项目折叠到列表底部
独立分组，opacity-60，默认折叠。

### 影响范围

- 新建：`apps/desktop/renderer/src/shell/ProjectContextMenu.tsx`
- 修改：`apps/desktop/renderer/src/shell/LeftSidebar.tsx` — 项目行加 onContextMenu / inline rename input / Archived 分组
- 修改：`packages/space-ipc-schema/src/channels/project.ts` — projectSchema 加 optional `archived` 字段；新 2 channels：
  - `project.recent.rename` (input: path + name → output: { renamed })
  - `project.recent.setArchived` (input: path + archived → output: { ok })
- 修改：`apps/desktop/electron/projects/store.ts` — 新方法 `rename`、`setArchived`；后者 archived=false 时 omit 字段（清洁序列化）
- 修改：`apps/desktop/electron/ipc/project.ts` — register 2 handler，走 `projectStore.assertAllowed`（path-probing 防御）

### 安全

- 两个 IPC handler 都走 `assertAllowed`（同 gitStats/fileSearch family）—— path 必须在 allowlist 才放行，
  防 renderer 拿任意 path 探测哪些在 allowlist。
- rename `name.trim()` 空字符串 reject；schema 卡 1..256 字符上限。
- Remove 走 `window.confirm` 二次确认（body text 明示"不动文件夹"）。

### 验收

- [x] 右键项目节点 3 项菜单（Rename/Archive/Remove）+ 键盘 R/A/D + Esc 关
- [x] Inline rename 用 input；blur=cancel, Enter=commit, Esc=cancel（review HIGH 双 fire 已修）
- [x] Archive 切归档，归档项目隐藏到 "Archived (N)" 折叠分组
- [x] Remove 走 confirm；取消时菜单仍可见（review MED-3 修）
- [x] 全部 IPC 失败有 toast 反馈，不静默 drop（review MED-4 修）
- [x] 11 个 ProjectStore 单测覆盖 rename/setArchived edge cases

## OC-21: ToolRegistry (extensible tool-card input rendering)

### 需求概述

把 `bubbles.tsx` 里 hardcoded `if (toolName === 'edit' || 'multi_edit' || 'write')` if-chain
重构成 registry-driven lookup。新工具加自定义渲染只需 `registerToolInputRenderer(toolName, fn)`，
不再要改 bubbles。

### 影响范围

- 新建：`apps/desktop/renderer/src/features/session/messages/toolRegistry.ts` — pure registry，`Map<toolName, fn>`
- 新建：`apps/desktop/renderer/src/features/session/messages/toolRenderers.tsx` — 内置 write/edit/multi_edit；side-effect import 即注册
- 修改：`bubbles.tsx` ToolEditInputView 改成 registry dispatch + 持 collapse state 统一 fallback

### 设计取舍

- Renderer 是 **pure function 返 `JSX.Element | null`**（不是 React.FC）
  - 返 null = "shape 不对" → caller 走 raw-JSON collapse fallback
  - 需要 hooks 的 renderer（multi_edit 的 showAllEdits）让返回 JSX 内嵌一个使用 hooks 的子组件（MultiEditList）
- 内置注册靠 import side effect；vite/rollup 不会 tree-shake bare side-effect import

### 验收

- [x] write / edit / multi_edit 输出跟以前 1:1 一致
- [x] 任意未注册工具（如 bash / read）现在走 raw-JSON collapse 视图（带 Show full / Collapse UX）
- [x] 7 个 registry 单测：lookup miss / register+lookup / overwrite / sorted list / 快照 immutable / clear / args shape
- [x] 删除冗余 `RawJsonInput` helper —— 改由 ToolEditInputView 顶部统一 fallback

## KX-I-05: 智能权限批处理 modal

### 需求概述

SDK 一个 turn 内可能同时请求多个 permission（multi_edit 给 N 个 file write 同时请求）。
原 PermissionModal 一次只显示队列头部一个，用户被 N 个连续 modal 轰炸。
KX-I-05：队列头部"同 sessionId + 非 danger"连续 ≥ 2 条时合并成 batch view，
顶部 Allow all (N) / Deny all (N) + 每行独立 Allow/Deny 兜底。

### 影响范围

- 新建：`apps/desktop/renderer/src/features/permission/permissionBatching.ts` —— pure 选择器
  `selectPermissionBatch(queue) → { mode: 'single', head } | { mode: 'batch', items, sessionId }`
- 修改：`PermissionModal.tsx` —— `useMemo(selectPermissionBatch(queue))`；batch 模式 render
  新 `PermissionBatchView` 子组件；single 模式走原代码路径

### 安全 / 设计约束

- **DANGER request 永远不入 batch** —— 必须单独走 typed-confirm，硬规则。10 个 selector 单测覆盖
  ("head is danger → single", "danger in middle stops batch", etc.)
- **答复用 Promise.all 并发**；main 端 PermissionBroker 单线程 + idempotent，安全
- **try/finally 包 setBusy(false)** —— IPC throw 不会让 modal 卡死（review HIGH 修）
- **`selection.mode === 'batch'` 时关掉 single-view keyboard listener** —— 否则 Esc 同时 fire 双 IPC（review LOW 修）

### 验收

- [x] 队列里 ≥ 2 同 session 非 danger → batch view
- [x] 队列里有 danger → 永远 single typed-confirm path
- [x] Allow all / Deny all 并发发 N 条 permission.answer；Promise.all 拒绝时 busy 仍回归 false
- [x] 每行独立 Allow / Deny 按钮兜底（用户想精细控制时用）
- [x] Esc=Deny all / Enter=Allow all once 键盘快捷
- [x] 10 个 selector 单测
