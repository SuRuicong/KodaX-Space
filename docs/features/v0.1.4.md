### 实现步骤

### 验收标准

## FEATURE_026: ⌘Shift+P 命令面板（JS fuzzy MVP）

### 需求概述

> 2026-06-05 audit：原 plan 里的 @-mention 在 v0.1.x 已落地（`AtPathPopover`，substring 排序）；slash command popover 在 F031 已做。
> 真正缺的就 **全局 ⌘Shift+P 命令面板** 一项；@-mention 的 fuzzy 升级（fzf 替 substring）作为本 feature stretch goal。

> 2026-06-05 拆分：原 F026 NAPI native-fuzzy 拆两层：
> - **本 feature（UI 层）**：⌘Shift+P 全局命令面板，用 JS fuzzy 做 MVP
> - **F042（native 层）**：等性能真痛了再升级 napi-rs

⌘Shift+P 全局唤出 modal，搜：
- **Actions**：New session / Switch project / Toggle theme / Open settings / Toggle dev tools …
- **Sessions**：当前项目下最近 sessions（点击切过去）
- **Files**：当前项目所有文件（点击插入 `@file_path` 到 input）
- **Slash commands**：已注册 slash 命令清单（点击装到 input 待执行）

形态对标：VS Code Ctrl+P / Cmd+Shift+P、Linear ⌘Shift+P、Raycast。

### 影响范围

- 新建：`apps/desktop/renderer/src/shell/CommandPalette.tsx` — modal + 4 group 渲染 + 键盘导航
- 新建：`apps/desktop/renderer/src/shell/commandSources.ts` — action / session / file / slash 候选生成 + 派发
- 新建：`apps/desktop/renderer/src/lib/fuzzy.ts` — FuzzyMatcher 抽象 + JS fzf-lite 实现
- 修改：`apps/desktop/renderer/src/shell/Shell.tsx` — 全局 keydown listener 注册 ⌘Shift+P / Ctrl+Shift+P，注入 `<CommandPalette />`
- 修改：`apps/desktop/renderer/src/store/appStore.ts` — 加 `commandPaletteOpen` + `toggleCommandPalette()`
- 修改（stretch）：`apps/desktop/renderer/src/shell/AtPathPopover.tsx` — 接 `lib/fuzzy.ts` 替换 substring rank
- 复用：`project.fileSearch`、`slash.discover`、`session.list` 三个已有 IPC，**0 新增 channel**

### 技术方案

**Fuzzy 抽象层**（让未来 F042 native 切换无痛）：

```typescript
// apps/desktop/renderer/src/lib/fuzzy.ts
export interface FuzzyMatcher {
  setCandidates(items: readonly string[]): void;
  search(query: string, limit?: number): readonly { item: string; score: number }[];
}
export function createMatcher(): FuzzyMatcher; // 默认 JS；F042 后切 native
```

**⌘Shift+P 命令面板形态**：
- Modal overlay（背景半透明、ESC 关、点外面关）
- 顶部单 input（autoFocus），4 分组列表（Actions / Sessions / Files / Slash），上下箭头跨组移动
- Enter 触发选中项的 `onPick()`；不同候选有不同语义：
  - Action → 调 store action（toggleTheme 等）
  - Session → setCurrentSession(id)
  - File → 插入 `@file_path` 到 BottomBar input（与 @-mention 结果一致）
  - Slash command → 把 `/cmd ` 字串插 input + focus

**全局键盘**：在 Shell.tsx top-level useEffect 注册 `keydown`，匹配 `(e.metaKey||e.ctrlKey) && e.key === 'k'` 时 preventDefault + toggle。**避开 input 焦点冲突**：当前焦点在 textarea/input 且不是修饰键组合时不抢。

**Modal 焦点管理**：开时 input.focus()，关时还焦点回 BottomBar textarea（不破坏用户原来在打的字）。

### 接口契约

```typescript
// commandSources.ts
export type CommandKind = 'action' | 'session' | 'file' | 'slash';
export interface CommandItem {
  readonly id: string;          // 稳定 key
  readonly kind: CommandKind;
  readonly label: string;       // 显示
  readonly hint?: string;       // 副标题（session 时显示项目路径）
  readonly searchText: string;  // fuzzy 输入串
  readonly onPick: () => void | Promise<void>;
}
export async function gatherCommands(ctx: {
  projectPath: string | null;
  sessionId: string | null;
}): Promise<readonly CommandItem[]>;
```

### 实现步骤

1. lib/fuzzy.ts FuzzyMatcher 抽象 + JS fzf-lite 实现 + 单测（exact / partial / acronym / 排序稳定性）
2. commandSources.ts：4 个 kind 的候选生成（action 是 const，session/file/slash 走 IPC）
3. CommandPalette.tsx：modal + input + grouped list + keyboard nav
4. Shell.tsx：global keydown 注册 + state 接 store
5. appStore：commandPaletteOpen + actions
6. e2e: 唤出 / 上下箭头 / Enter 4 种 kind / Esc 关 / 焦点回还
7. （stretch）AtPathPopover 接 fuzzy lib 替 substring

### 验收标准

- [ ] ⌘Shift+P (macOS) / Ctrl+Shift+P (Win/Linux) 唤出 < 100ms
- [ ] 4 group 都能搜（Actions / Sessions / Files / Slash），分组标题清晰
- [ ] 上下箭头跨组移动正确，Enter 触发对应行为
- [ ] Esc 关闭后焦点回 BottomBar textarea，不丢用户已打的字

### 不在本 feature 范围

## FEATURE_042: NAPI native-helpers（合并版）

> 2026-06-05 新立项：把 v0.1.4 plan 里的 F025（native-diff）+ F026 的 native fuzzy 部分合并；推到 v0.1.x 后续版本。
> 目标：等"实际场景测出 JS 实现的痛点"再启动，不做无症状的过早优化。

### 需求概述

把两个潜在的性能热点（diff 计算 / fuzzy 匹配）一并迁到 napi-rs：

- **native-diff**：大文件 unified diff 计算（imara-diff 或 similar）
- **native-fuzzy**：fzf-style 模糊匹配（nucleo / fzf-rs）

### 启动条件（必须满足才动）

- 用户实际反馈或 benchmark 数据证明 JS 实现达到下面任一阈值：
  - 文件 diff 计算耗时 > 200ms（单次 file panel diff 渲染）
  - @-mention 弹下拉 P99 > 100ms（5k 文件项目）
  - ⌘Shift+P 命令面板搜索 P99 > 80ms
- 当前 F026 UI 上线后 1-3 个月内观察使用频度与延迟数据

### 影响范围（启动时再细化）

- 新建：`packages/native-helpers/`（napi-rs 项目，diff + fuzzy 同 crate）
- 修改：`apps/desktop/renderer/src/lib/fuzzy.ts` createMatcher() 切实现
- 修改：File panel diff 调用点切 native
- 新建：6 平台 CI 矩阵（同 [FEATURE_014](v0.1.1.md#feature_014-napi-native-tokenizer) pattern）

### 技术方案要点

- 一个 napi crate 同时 export `diff()` 和 `Matcher` class（减少 6 平台 binary 数量）
- 沿用 F014 native-tokenizer 的 build / 发布管道（已经成熟）
- Fallback：当前 JS 实现保留，native 加载失败时透明降级

### 验收标准（待启动时定）

性能数字由 F026 上线后实测数据回填，避免无目标拍脑袋。

### 不在本 feature 范围

- 编辑器 / 终端 / 文件预览的性能（不是 fuzzy/diff 算法热点）
- 跨进程 IPC overhead（设计层优化，不是 napi 解决的事）

## FEATURE_040: 多项目可折叠 session 侧栏 + per-session 状态指示

### 需求概述

当前 [LeftSidebar](../../apps/desktop/renderer/src/shell/LeftSidebar.tsx) 只认单一 `currentProjectPath`，session 列表是扁平的「当前项目」视图，且没有 per-session 实时状态——用户看不出哪个 session 正在跑任务、哪个在等他确认。

对标 **OpenAI Codex 桌面客户端**（IDE 风 `project sidebar → threads/sessions` 层级，每个 thread = 一个 agent 实例，关掉重开 session 全保留）升级左侧栏为：

1. **多项目可折叠树**：顶层列出所有最近项目，每个可 `▸/▾` 折叠，当前项目默认展开 + 高亮。
2. **session 历史折叠**：每项目展开后默认只显示前 N 条（如 5），其余「还有 X 条…」可展开——长历史不撑爆侧栏。
3. **per-session 状态点（仅左栏，不做标签页）**：🟢 运行中 / 🟡 等待用户确认 / 🔴 错误 / 无点=idle。折叠的项目聚合显示运行计数（`🟢2`），一眼看到哪个项目有 agent 在跑。

> 形态决策（用户拍板 2026-06-03）：**只在左栏显示状态点，不引入对话区上方的标签页**——符合「极简且智能」哲学（codex 至今也只有 sidebar、标签页是其社区未满足的需求 [openai/codex#10886](https://github.com/openai/codex/issues/10886)）。多项目与状态都从已有数据**自动呈现**，用户零额外操作。

### 影响范围

- **已有可复用**：
  - [projects/store.ts](../../apps/desktop/electron/projects/store.ts) `ProjectStore` —— 多项目列表数据层（`~/.kodax/space/projects.json`，path/name/lastUsedAt）已存在，多项目地基已就绪。
  - `buildSessionTreeOrder` / `SessionTree`（LeftSidebar.tsx）—— fork 树渲染，作为「项目内 session 列表」直接复用。
  - `RunningPeersPanel`（跨进程 `session.listRunning`）—— 与本 feature 的「本进程内 per-session 状态」互补，不冲突。
- **main 修改**：projects IPC（list/add/remove/setCurrent，若缺补全）；新增「本进程内多 session 状态投影」（track 每个 session 是 running / awaiting / error）。
- **renderer 重构**：LeftSidebar.tsx（单项目 → 多项目树外层 + 项目内 SessionTree）；appStore（`currentProjectPath` 旁加项目列表、折叠状态、跨项目 session 状态 map）。

> 注意：本 feature 与正在进行的 LeftSidebar 改动有重叠，落地时以届时的 LeftSidebar 现状为基线对齐。

### 技术方案

- **项目树**：`ProjectStore.list()` → 顶层项目节点，按 `lastUsedAt` 倒序；`currentProjectPath` 对应节点高亮 + 默认展开。折叠/展开状态持久化到 localStorage（key 如 `kodax-space.expandedProjects`）。
- **项目内 session**：项目展开时 `session.list({ projectRoot })` 拉该项目 sessions，复用 `SessionTree`（fork 树 + pinned/archived/unread + 现有 filter）。默认渲染前 N 条，超出折叠为「还有 X 条…」按钮，点开显示全部。
- **per-session 状态枚举**：`idle | running | awaiting | error`。
  - 当前 active session：复用 appStore 已有的 streaming 事件（running）+ askUser 队列（awaiting）+ 错误事件（error）。
  - 后台 / 其他项目 session：main 维护「活动 session 状态表」，随 SessionEvent 变更 push 给 renderer（或低频轮询投影），renderer 存进跨项目 `sessionStatus` map。
  - 渲染：session 行尾状态点；项目节点聚合该项目下 running session 计数（`🟢N`），折叠时也显示。
- **不做标签页**：对话区上方不加 tab bar。
- **「极简且智能」自检**：多项目自动来自 ProjectStore、状态自动来自事件流——不增加用户操作负担；无新增配置项。

### 接口契约

```typescript
// projects IPC（已有则复用，缺则补）
'projects.list'       → { projects: { path: string; name: string; lastUsedAt: number }[] }
'projects.add'        → { path } → { ok: boolean; project }
'projects.remove'     → { path } → { ok: boolean }
'projects.setCurrent' → { path } → { ok: boolean }   // 切当前项目根

// per-session 状态投影（二选一）
'session.statusList'  → { statuses: { sessionId: string; status: 'idle'|'running'|'awaiting'|'error' }[] }
// 或：在现有 SessionEvent push 流里增加 status 变更事件，renderer 增量更新 sessionStatus map
```

### 实现步骤

1. main：补全 projects IPC（list/add/remove/setCurrent）；实现本进程内 session 状态表（running/awaiting/error 来源于现有 KodaXEvents + askUser）。
2. appStore：加项目列表 state、`expandedProjects` 折叠 state（localStorage 持久化）、跨项目 `sessionStatus` map。
3. LeftSidebar：新增 `ProjectTree` 外层（项目折叠节点）包裹现有 `SessionTree`；当前项目高亮 + 默认展开。
4. session 历史折叠：前 N 截断 + 「还有 X 条…」展开。
5. per-session 状态点 + 项目级运行计数聚合。
6. 持久化：展开项目集合、当前项目。
7. 测试：ProjectTree 渲染与折叠、状态枚举映射、前 N 截断、当前项目高亮、切项目加载对应 session。

### 验收标准

- [ ] 顶层列出所有最近项目（来自 ProjectStore），当前项目默认展开 + 高亮
- [ ] 项目可折叠/展开，折叠状态重启后恢复
- [ ] 每项目 session 历史默认前 N 条 + 「还有 X 条…」可展开全部
- [ ] 运行中 session 显 🟢、等待确认显 🟡、错误显 🔴、idle 无点
- [ ] 折叠的项目显示运行计数（🟢N），一眼看到哪个项目有 agent 在跑
- [ ] 不引入对话区标签页（仅左栏状态点）
- [ ] 切换项目正确加载对应 session 列表，互不串台
- [ ] 单测覆盖 ProjectTree 折叠 / 状态映射 / 前 N 截断 ≥ 80%

## FEATURE_041: 右侧栏改造为「任务态 mission control」（Plan / Workers / Changes）+ 退役 StashNotice

### 需求概述

当前对话区底部上方挂着一条 [StashNotice](../../apps/desktop/renderer/src/shell/StashNotice.tsx) 横幅（`● Uncommitted: 8 modified · 1 staged on main`）。它占用主对话列的纵向空间、信息量低（只有计数），且把「git 变动」这类**周边态信息**塞进了用户的阅读主轴。

与此同时，「当前任务正在发生什么」被拆散在**两个互不相干的表面**：

- [RightSidebar](../../apps/desktop/renderer/src/shell/RightSidebar.tsx) 有 Progress（todo）/ Working folder / Context 三节，常驻但只读摘要；
- [CommandToolbar](../../apps/desktop/renderer/src/shell/CommandToolbar.tsx) 的 `▥` 下拉里 [PlanPanel](../../apps/desktop/renderer/src/shell/popouts/PlanPanel.tsx) / [TasksPanel](../../apps/desktop/renderer/src/shell/popouts/TasksPanel.tsx)（worker tree）是**盖住整个对话区的 popout overlay**，要主动呼出、看完再关。

结果是 Plan **被渲染了两遍**（RightSidebar.ProgressSection 与 PlanPanel 同源 `todoListBySession`），而子 Agent 状态只在 overlay 里、平时不可见。

本 feature 把右侧栏重塑为常驻的「任务态 mission control」，回答三个问题——**KodaX 打算做什么 / 正在做什么 / 改了什么**：

1. **Plan（计划列表）**：KodaX Scout 派的 todo list。
2. **Workers（子 Agent 工作和状态）**：worker tree（标题 + 状态点 + phase + budget）。
3. **Changes（项目目录 git 变动）**：改动**文件列表** + `M`/`A`/`U` 徽标，点文件 → 看该文件 diff。**取代 StashNotice 横幅**。

> 形态决策（用户拍板 2026-06-04）：
> - **Plan / Workers 常驻摘要 + 点标题弹大图**：右侧栏放紧凑摘要，点 section 标题（`⤢`）才弹原 overlay 看完整细节。`▥` 下拉**移除** Plan / Tasks 两项（避免两个入口重复），保留 Preview / Diff / Terminal / Agents / MCP。
> - **Changes 只读 + 点击看 diff**：文件列表点行 → DiffPanel 看该文件 diff；**不做** 一键 commit/stash/discard（保持只读，避免引入 git 写操作 IPC + 安全面）。
> - **旧两节降级**：Working folder / Context 保留但默认折叠、置于底部；新三节置顶默认展开。
> - **「极简且智能」自检**：三节数据全来自已有事件流（todoList / managed_task_status / git status）——零新增用户操作、零新增配置项；信息从「要主动呼出 overlay」升级为「常驻自动呈现」。

### 影响范围

- **已有可复用**：
  - `PlanPanel` 渲染 → Plan 节直接复用（并**删除** RightSidebar 现 `ProgressSection` 这处重复渲染）。
  - `TasksPanel` 的 [buildWorkerTree](../../apps/desktop/renderer/src/shell/popouts/worker-tree.ts) + `WorkerRow` → 抽成共享件，侧栏（紧凑）与 Tasks overlay（完整）**单一来源**，杜绝 Plan 那种双份。
  - [DiffPanel](../../apps/desktop/renderer/src/shell/popouts/DiffPanel.tsx) + `MonacoDiffViewer` → Changes 点文件复用。
  - `requestPopout` store action（Shell 已消费）→ section 标题 `⤢` 触发「弹大图」。
- **main 修改**：
  - 新增 `project.gitChanges` channel（见接口契约）——返回**带路径的**变动文件列表，区别于现有只回计数的 `project.gitStatus`。
  - 扩展 per-file diff 来源：现 `files.diff` 仅命中 write/edit 工具调用缓存，外部编辑器改的文件会 miss；加 git fallback（`git show HEAD:<path>` 当 before、工作区文件当 after），让任意变动文件可 diff。
- **renderer 重构**：
  - `RightSidebar.tsx` 重排为 Plan / Workers / Changes（默认展开）+ Working folder / Context（默认折叠、置底）。
  - `CommandToolbar.tsx` 的 `POPOUTS` 移除 `tasks` / `plan`。
  - `BottomBar.tsx` 移除 `StashNotice` 挂载；退役该组件（其 debounce / focus / 30s 兜底刷新逻辑迁入 Changes 节或共享 hook）。

> ⚠️ **逆转既有设计决策（需 security-reviewer 把关）**：`project.gitStatus` 的 schema 注释明确写「**不**返回文件路径列表（DoS guard + 隐私）」。本 feature 因要渲染文件列表而新增带路径的 `project.gitChanges`。缓解：(1) renderer 是同进程本地 Electron 宿主，"隐私 leak 到 renderer" 风险弱；(2) 保留 DoS guard——文件列表**上限 200 条** + `truncated` 标志，路径长度上限 2048；(3) 复用 `validateProjectRoot` + `runGit`（shell:false / 5s 超时 / 输出截断）；(4) 轻量 `gitStatus` 计数版**保持不变**，不影响既有调用方。

### 技术方案

- **Plan 节**：复用 `PlanPanel` 主体；header 显示 `done/total` + active `activeForm` 摘要，`⤢` → `requestPopout('plan')`。
- **Workers 节**：从 `TasksPanel` 抽 `buildWorkerTree` + `WorkerRow` 为共享渲染；侧栏只展 active worker + 计数摘要 + budget 进度条，`⤢` → `requestPopout('tasks')` 看完整树。
- **Changes 节**：
  - 调 `project.gitChanges({ projectRoot })` 拿文件列表；沿用 StashNotice 的刷新触发器（切 project 立刻拉 + write/edit/bash tool_result debounce 800ms + window focus / visibilitychange + 30s 兜底；main 端 5s TTL cache 防 hammer）。
  - 渲染：每行 `<badge M/A/U> <path>`，点行 → `requestPopout('diff', { path })`，DiffPanel 接住 path 渲染该文件 diff（命中工具缓存走缓存，否则走 git fallback）。
  - 空 / 非 git repo → 该节渲染「working tree clean」一行或不渲染。
- **section 折叠 + 自动展开**：复用现 `Section` 组件；Plan / Workers / Changes `defaultOpen`，有 live 内容时确保展开；Working folder / Context `defaultOpen=false`。
- **退役 StashNotice**：删 BottomBar 挂载点 + 组件文件；其计数信息被 Changes 节文件列表上位替代。

### 接口契约

```typescript
// 新增 project.gitChanges —— 带路径的变动文件列表（区别于只回计数的 project.gitStatus）
'project.gitChanges'  → { projectRoot: string }
  → {
      isGitRepo: boolean;
      branch: string | null;
      files: {
        path: string;          // 相对 projectRoot 的 posix 路径，max 2048
        status: 'M'|'A'|'D'|'R'|'U';  // U = untracked
        staged: boolean;
      }[];                     // 上限 200
      truncated: boolean;      // 变动超 200 时 true，UI 显示 "+N more"
    }

// 扩展 files.diff —— 工具缓存 miss 时 git fallback（schema 不变，行为增强）
//   before = `git show HEAD:<path>`（untracked → 空串）, after = 工作区文件内容
```
