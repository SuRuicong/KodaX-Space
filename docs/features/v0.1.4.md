## FEATURE_023: 内置终端多 tab

### 需求概述

[FEATURE_011](v0.1.1.md#feature_011-内置终端xtermjs--node-pty-单-tab) 单 tab 升级到多 tab。每 tab 独立 cwd / pty 进程。可改名、重排。

### 影响范围

- 修改：`apps/desktop/renderer/features/terminal/TerminalPane.tsx`
- 修改：`apps/desktop/electron/ipc/terminal.ts`（多 pty 管理）

### 技术方案

- 一个 pty 池（map<ptyId, NodePtyProcess>）
- 每 tab 一个 pty + 独立 xterm 实例
- Tab 头：名 / cwd 提示 / 关闭按钮 / + 按钮
- 切 tab 时只 attach 不杀进程
- "New tab in current session cwd" 快捷动作
- 拖拽 tab 重排

### 接口契约

```typescript
// 复用 v0.1.1 的 IPC，加：
'terminal.list' → { ptys: { id, cwd, name }[] }
'terminal.rename' → { ptyId, name } → { ok }
```

### 实现步骤

1. PtyPool 管理器
2. xterm 多实例 + tab 切换
3. Tab UI
4. 重排（react-dnd 或纯 onMouseDown）
5. cleanup：关 tab 杀对应 pty
6. 数量上限（如 10），超出提示

### 验收标准

- [ ] 同时开 5 个 tab 不互相阻塞
- [ ] 切 tab 后输出历史保留
- [ ] 关 tab 后无僵尸 pty
- [ ] 命名 / 重排 持久化（重启 Space 后恢复）

## FEATURE_024: 文件富预览（PDF / docx / xlsx）

### 需求概述

File panel 中点击非代码文件时，渲染对应预览，不只是显示 "binary file"。

### 影响范围

- 新建：`apps/desktop/renderer/features/code/PdfPreview.tsx`
- 新建：`apps/desktop/renderer/features/code/DocxPreview.tsx`
- 新建：`apps/desktop/renderer/features/code/XlsxPreview.tsx`
- 新建：`apps/desktop/electron/ipc/preview.ts`

### 技术方案

- PDF：用 `pdf.js` 在 renderer 渲染
- docx：用 `mammoth` 转 HTML 在 renderer 显示
- xlsx：用 `sheetjs` (`xlsx`) 解析后渲染成 HTML 表格
- pptx：v0.1.4 不做（后续）
- 大文件分块：> 10MB 拒绝预览
- 预览只读；用户编辑请回 IDE

### 接口契约

```typescript
'preview.canPreview' → { path } → { kind: 'pdf'|'docx'|'xlsx'|'image'|'video'|'binary' }
'preview.docx.toHtml' → { path } → { html: string }
'preview.xlsx.toJson' → { path, sheet?: string } → { sheets, data }
// PDF 直接 renderer 读取 blob
```

### 实现步骤

1. 装 pdf.js / mammoth / sheetjs
2. 文件 kind 探测
3. 各 preview 组件
4. main 提供 file blob read（限制 path）
5. 错误兜底（损坏文件、加密 PDF 等）

### 验收标准

- [ ] PDF 翻页流畅，缩放可用
- [ ] docx 渲染保留标题 / 列表 / 表格基本结构
- [ ] xlsx 多 sheet 切换，数据正确
- [ ] 损坏文件不崩 Space
- [ ] > 10MB 文件显示拒绝提示

## FEATURE_025: NAPI native-diff

### 需求概述

第二个 Rust 加速器：大文件 diff 计算用 `imara-diff`（rust-analyzer 同款），替换 JS 实现。

### 影响范围

- 新建：`packages/native/diff/`
- 修改：File panel diff 计算调用点

### 技术方案

- napi-rs 项目，依赖 `imara-diff = "0.1"` 或 `similar = "2"`
- API：`diff(before: string, after: string, opts?: DiffOpts) → DiffHunk[]`
- 输出 unified diff hunks，含 line numbers
- 6 平台 build matrix（同 [FEATURE_014](v0.1.1.md#feature_014-napi-native-tokenizer)）
- Fallback：JS 实现（diff 包）

### 接口契约

```typescript
// @kodax-ai/native-diff
export type DiffHunk = { startBefore, countBefore, startAfter, countAfter, lines: DiffLine[] };
export type DiffLine = { kind: 'context' | 'added' | 'removed', text: string };
export function diff(before: string, after: string, opts?: DiffOpts): DiffHunk[];
```

### 实现步骤

1. 创建 napi-rs 项目
2. imara-diff 集成 + JSON 序列化
3. 6 平台 CI
4. npm 发布（同 native-tokenizer pattern）
5. File panel diff 集成 + fallback
6. Benchmark vs JS 实现（10MB / 100MB 文件）

### 验收标准

- [ ] 10MB 文件 diff < 200ms（vs JS ~2s）
- [ ] 100MB 文件 diff < 5s（JS 可能超时）
- [ ] 输出与 git diff 一致（采样验证）
- [ ] Fallback path 在 NAPI 不可用时工作

## FEATURE_026: NAPI native-fuzzy

### 需求概述

第三个 Rust 加速器：@-mention 文件选择器 + 命令面板的模糊搜索用 `nucleo`（Helix 同款）。

### 影响范围

- 新建：`packages/native/fuzzy/`
- 新建：@-mention UI（Code 面板输入框中触发）
- 新建：命令面板（⌘K）

### 技术方案

- napi-rs 项目依赖 `nucleo = "0.5"`
- API：`createMatcher() / addCandidates(items) / search(query) → ScoredMatch[]`
- 用法：项目 10k+ 文件名 + 符号名预 index，search 实时调用
- @-mention：输入框打 `@` 触发，弹下拉，箭头键选
- 命令面板：⌘K 唤出，搜 action / file / session

### 接口契约

```typescript
// @kodax-ai/native-fuzzy
export class Matcher {
  addCandidates(items: string[]): void;
  search(query: string, limit?: number): { item: string, score: number }[];
  clear(): void;
}
```

### 实现步骤

1. 创建 napi-rs 项目
2. nucleo 封装
3. 6 平台 build + 发布
4. @-mention UI + Matcher 集成
5. 命令面板 UI（CommandPalette 组件）
6. Benchmark：10k 文件、10k 符号、查询 P99 < 50ms

### 验收标准

- [ ] @-mention 在 10k 文件项目中 P99 < 50ms
- [ ] 命令面板 ⌘K 唤出 < 100ms
- [ ] 搜索结果与 fzf 行为一致（采样验证 fuzzy ranking）
- [ ] 大项目 index 完成耗时 < 2s（10k 文件）
