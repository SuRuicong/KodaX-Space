## Batch Intent

`KodaX owns runtime behavior; Space owns visibility, review, recovery, and user control.`

KodaX is adding durable workflow replay resume, generate-without-run drafts, and pipeline (fan-out/fan-in) execution. v0.1.31 turns those runtime capabilities into a desktop workflow host: Space must show replay/recovery state, make draft review legible before a workflow starts, and render pipeline runs with time/phase semantics instead of a tree-only view.

The timeline and state reads SDK snapshots/events only; Space never infers workflow semantics from transcript text.

## FEATURE_085: Workflow Durable Replay Resume UI

### 需求概述

KodaX is adding durable workflow replay resume. Space needs to show replay/recovery state instead of treating restarted workflows as ordinary fresh runs. After an app restart, resumable runs must be discoverable and actionable.

### 影响范围

- Renderer workflow panel (run list + run detail).
- Session/workflow run state model consumed from SDK snapshots/events.
- Transcript rendering must not duplicate replayed child-agent activity.

### 技术方案

Add workflow run states and actions:

- `resumable`,
- `replaying`,
- `replay-complete`,
- `resume-failed`,
- `orphaned`.

Workflow panel should show:

- why a run is resumable,
- last persisted event time,
- expected replay window,
- resume button,
- discard/archive action,
- divergence or missing-artifact warnings.

### 接口契约

- Consumes SDK durable-workflow snapshot/replay events (KodaX workflow v2 surface).
- No new Space-owned runtime truth; Space renders SDK-reported run state only.

### 实现步骤

_待 Plan 阶段细化（depends on the published KodaX workflow v2 replay snapshot contract）._

### 验收标准

- After app restart, resumable workflow runs are discoverable.
- Users can resume or discard from Space.
- Replay progress is visible and does not duplicate child-agent activity in the transcript.
- Failed replay gives a precise reason and next action.

## FEATURE_086: Workflow Draft Review and Approval UI

### 需求概述

Generated workflows are executable plans. KodaX will support generate-without-run and review-before-start; Space should make that review legible before a workflow starts, so natural-language workflow creation can stop at a reviewable draft state.

### 影响范围

- New draft workflow review panel in the renderer.
- Workflow launch path (must gate trusted-local execution behind approval).
- Draft persistence and provenance.

### 技术方案

Add a draft workflow review panel:

- intent summary,
- required inputs,
- capability requirements,
- tool and permission envelope,
- file/worktree impact,
- generated capsule metadata,
- risk and trust notes,
- approve/run, save draft, reject, or ask agent to revise.

Do not expose raw generated script as the primary UI, but include an expandable technical view for advanced users.

### 接口契约

- Consumes the KodaX generate-without-run / draft workflow capsule contract.
- Approval action maps to the SDK workflow start/preflight entry point.

### 实现步骤

_待 Plan 阶段细化（depends on the KodaX draft-capsule metadata contract）._

### 验收标准

- Natural-language workflow creation can stop at draft state.
- User approval is required before trusted-local workflow execution.
- Draft revisions preserve provenance.
- Saved drafts can be launched later with preflight.

## FEATURE_087: Workflow Pipeline Timeline and Replay-Aware Rendering

### 需求概述

Current workflow UI is tree-first. Pipeline workflows and replayed runs need time/phase semantics: stage order, fan-out/fan-in, skipped stages, and replayed events.

### 影响范围

- Renderer workflow panel: add a pipeline timeline mode beside the existing tree.
- Stage/child-agent/artifact cross-navigation.

### 技术方案

Add a pipeline timeline mode beside the existing tree:

- horizontal or vertical stage timeline,
- fan-out/fan-in grouping,
- per-stage status and elapsed time,
- token and child-agent counts,
- replay badges for restored events,
- artifact output markers.

The timeline reads SDK snapshots/events only; Space does not infer workflow semantics from transcript text.

### 接口契约

- Consumes SDK pipeline-stage snapshots/events (stage order, fan-out/fan-in, replay markers).

### 实现步骤

_待 Plan 阶段细化._

### 验收标准

- Pipeline workflows are understandable without opening raw logs.
- Replay events are visually distinct from newly emitted events.
- Users can jump from a stage to child-agent detail, artifact, or transcript evidence.
