> Target: 2026-Q3 中
> Theme: Repointel + session lineage + CLI ↔ Space teleport + Quick Ask
> Status: Historical rebaseline. F016 is completed; F015/F017/F018 remain Partial historical rows and are carried forward by F082/F084/F083 in v0.1.20, with SDK/CLI-gated remainders parked in F112 at v0.2.9.

> Definition of Done: Space 用户能享受 KodaX 生态全部独有能力——Repointel 仓库智能、session 树、跨 surface 漂移、全局问答

## 版本范围

v0.1.2 把 KodaX 生态的几个**只 KodaX 才有**的能力接入 Space：

- **Repointel**：KodaX-private 的仓库智能，状态条 + 一键 warm
- **Session lineage**：KodaX 已有的可分叉 session 树可视化
- **Cross-surface teleport**：CLI 与 Space 互相漂移
- **Quick Ask**：全局热键浮窗，桌面 app 的"轻便入口"

这是 KodaX Space **vs Claude Desktop / Codex Desktop 的差异化层**。

## 验收标准（版本级）

- [ ] 已安装 Repointel 时 Space 状态条显示 `premium-native ●`，一键 warm 可用
- [ ] 复杂 session（5+ checkpoint）能在 lineage 视图回放、从某节点分叉
- [ ] CLI 跑 `kodax --teleport space` 后 Space 收到通知、确认可接管
- [ ] Quick Ask 全局热键秒级唤出，回答完关闭无残留

## FEATURE_016: Session lineage 图

### 需求概述

KodaX 已有 branchable session tree。Space 把这棵树画成图，让用户回放某个 checkpoint、从某节点开新 branch。

### 影响范围

- 新建：`apps/desktop/renderer/features/session/LineageView.tsx`
- 新建：`apps/desktop/electron/ipc/lineage.ts`
- 调用：`@kodax-ai/session-lineage`（KodaX 包）

### 技术方案

- KodaX 内部已有 session-lineage 树（节点 = checkpoint，边 = continuation）
- 用 D3 或 react-flow 渲染树状/DAG
- 节点信息：checkpoint id / 创建时间 / 该节点 prompt 简述 / token 累计
- 操作：点节点 → 预览；右键 → "Branch from here" / "Revert to here"
- 入口：session 卡片菜单 → "View lineage"
- 不主导主视图——是次要工具，按需打开

### 接口契约

```typescript
'lineage.get' → { sessionId } → { nodes: LineageNode[], edges: LineageEdge[] }
'lineage.branchFrom' → { sessionId, checkpointId } → { newSessionId }
'lineage.revertTo'   → { sessionId, checkpointId } → { ok }
```

### 实现步骤

1. 接 `@kodax-ai/session-lineage` API
2. 用 react-flow 画图（节点 + 边）
3. 节点预览 hover card
4. Branch 操作（KodaX 内核已有）
5. Revert 操作（带二次确认）
6. 大 session（100+ checkpoint）的 virtualize / pan-zoom

### 验收标准

- [ ] 50+ checkpoint 渲染流畅
- [ ] Branch 后立刻在 SessionList 出现新 session
- [ ] Revert 操作不影响其他 branch
- [ ] 节点 hover 预览 < 100ms

## FEATURE_017: CLI ↔ Space 文件级 teleport

> Rebaseline：原 F017 的 end-to-end CLI ↔ Space 双向漂移没有在 v0.1.2 交付；v0.1.20 的 F084 已交付 Space-side handoff receiver/inbox/accept/dismiss。当前设计源是 [v0.1.20.md#slice-4-f084-space-side-handoff-receiver](v0.1.20.md#slice-4-f084-space-side-handoff-receiver)；CLI/REPL writer 与 Space→CLI pickup 的上游/后续集成挂到 [v0.2.0.md#feature_112-legacy-continuity-remainder-hooks](v0.2.0.md#feature_112-legacy-continuity-remainder-hooks)。

### 需求概述

实现 PRD §4.1 第 5 条承诺："One session, many surfaces"——CLI 的 session 可漂移到 Space，Space 的 session 可踢回 CLI。

### 影响范围

- 新建：`apps/desktop/electron/teleport-watcher.ts`
- 新建：`apps/desktop/renderer/features/session/TeleportPrompt.tsx`
- 协作：KodaX CLI 加 `--teleport space` 标志（KodaX 内核侧改动）

### 技术方案

- 文件系统 handoff（不走 ACP）：
  - `~/.kodax/handoffs/<sessionId>.json` 描述要漂移的 session
  - JSON 内容：`{ from: 'cli'|'space', sessionId, cwd, provider, mode, lastMessageId, pid, createdAt }`
- Space main 启动时 + 周期性 watch `~/.kodax/handoffs/`
- 发现新 handoff → 桌面通知 + UI 弹 "Continue session abc123 from terminal?"
- 用户确认 → main 调 `KodaXClient.resume(sessionId)` → 删除 handoff 文件
- Space → CLI：UI 按钮 "Continue in terminal" → main 写 handoff → 起 OS terminal 跑 `kodax --pickup <sessionId>`
- 并发锁：`~/.kodax/sessions/<id>.lock` 含 pid，写者必须持锁

### 接口契约

```typescript
'teleport.handoffs.list' → { handoffs: HandoffFile[] }
'teleport.accept'        → { sessionId } → { ok }
'teleport.reject'        → { sessionId } → { ok }
'teleport.toCli'         → { sessionId } → { ok }

// 事件
{ kind: 'teleport_incoming', handoff: HandoffFile }
```

### 实现步骤

1. KodaX 内核侧加 `--teleport space` 与 `--pickup <id>` 标志
2. Space main 用 `chokidar` watch `~/.kodax/handoffs/`
3. 文件锁逻辑（`fs.open` with `wx` flag）
4. TeleportPrompt UI（toast / dialog）
5. 跨 OS terminal 启动：macOS `open -a Terminal`、Win `wt.exe`、Linux 探测 `$TERMINAL`
6. 并发场景测试（CLI 写时 Space 不去 resume，反之亦然）

### 验收标准

- [ ] CLI `--teleport space` 退出 → Space 5s 内收到通知
- [ ] 接管后 conversation 历史完整、provider/mode 一致
- [ ] Space → CLI 起新 terminal 跑 pickup 成功
- [ ] 并发写两端不会破坏 JSONL（一方退）
- [ ] Handoff 文件 5 分钟未消费自动过期

## FEATURE_018: Quick Ask popover

> Rebaseline：原 F018 的「无 session、无工具、无持久化」语义没有按 v0.1.2 直接交付；v0.1.20 的 F083 已承接为 Quick Ask v2 continuity：临时 session 生命周期可见、可 Continue in Coder。当前设计源是 [v0.1.20.md#slice-3-f083-quick-ask-continuity-before-true-side-query](v0.1.20.md#slice-3-f083-quick-ask-continuity-before-true-side-query)；真正 no-session sideQuery 和 Partner promotion 挂到 [v0.2.0.md#feature_112-legacy-continuity-remainder-hooks](v0.2.0.md#feature_112-legacy-continuity-remainder-hooks)。

### 需求概述

全局热键唤出独立浮窗，问临时问题。无 session、无工具、无持久化（[ADR-004](../ADR/ADR-004-panel-model.md)）。

### 影响范围

- 新建：`apps/desktop/renderer/features/quick-ask/QuickAskWindow.tsx`
- 新建：`apps/desktop/electron/quick-ask-window.ts`
- 修改：main 注册全局快捷键

### 技术方案

- macOS `⌘⇧K`、Win/Linux `Ctrl+Shift+K`（用户可改）
- main 用 `globalShortcut.register` 注册
- 唤出时创建/显示 `quickAskWindow`（frameless、always-on-top 一次、约 480×360px、屏幕中心）
- Renderer 跑一个独立 React 入口（vite multi-entry）
- 后端：在 main 进程创建临时 KodaXClient，`session: undefined`、强制 mode = 'plan'、`mcpServers: []`
- 关闭：失焦 / Esc → 销毁 KodaXClient（不调 storage.save）
- 多聊：底部按钮 "Continue in Code panel as new session" → 转 Code session（此时落盘）

### 接口契约

```typescript
'quickAsk.send' → { content, providerId } → { tempId }
'quickAsk.close' → { tempId } → void
'quickAsk.continueInCode' → { tempId } → { newSessionId }

// 事件（main → quick-ask renderer）
{ kind: 'quickAsk.delta', tempId, text }
{ kind: 'quickAsk.complete', tempId }
{ kind: 'quickAsk.error', tempId, error }
```

### 实现步骤

1. 全局快捷键注册 + 冲突检测（如已被占用提示用户改）
2. 单独 BrowserWindow（frameless, transparent 可选）
3. Vite 多入口配置（main + quick-ask）
4. main 临时 KodaXClient 池（避免每次 spawn 开销）
5. "Continue in Code" 把对话历史 inject 到新 KodaXClient
6. Provider 选择：默认沿用上次 Quick Ask 选定的（写 `~/.kodax/space/quick-ask.json`）

### 验收标准

- [ ] 全局快捷键在所有应用前台时都可唤出
- [ ] 唤出 < 200ms
- [ ] 失焦自动收起
- [ ] 关闭后 session 不出现在 SessionList
- [ ] Continue in Code 后历史完整迁移到 Code 面板
- [ ] 不写 `~/.kodax/sessions/`（验证：发完关一次后该目录无新文件）
