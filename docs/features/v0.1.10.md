## FEATURE_044: 右侧 Changes 点文件打开 git working-tree diff popout

### 需求概述

用户报: 打开历史 session,右侧 RightSidebar 的 Changes 节点击文件想看 diff,但 DiffPanel 显示 "No diff available — tool call cache miss"。原因是当前 DiffPanel 走的是 `files.diff` IPC,只查 main 端 in-memory tool-call write/edit 缓存。历史 session 的 tool-call 没在本 Space 进程跑过,缓存空 → 永远 miss。

用户期望: **任何时候**点击 Changes 节里列出的文件 (本就是 git 检测到的"已改文件"),都能看到该文件**当前 working tree vs HEAD** 的左右对比 diff。这跟 tool-call 那一瞬的 before/after 是两条独立的语义:

- Tool-call cache diff (现有,内嵌在对话流的 ToolDiffView):"AI 改文件那一瞬"
- Git working-tree diff (本 feature):"现在工作目录 vs HEAD 提交"

两者对历史 session / 多次 edit 累积 / 用户手改 都有不同价值,**都需要保留**。

### 影响范围

- **新 IPC**: `project.gitFileDiff` (main → renderer)
  - input: `{ projectRoot: string, path: string }`
  - output: `{ available: boolean, before: string, after: string, isUntracked?: boolean, isBinary?: boolean }`
- **renderer**: `DiffPanel.tsx` 在 `files.diff` cache miss 时 fallback 调 `project.gitFileDiff`
- **schema**: `packages/space-ipc-schema/src/channels/project.ts` 加 channel 定义
- **renderer**: `RightSidebar.tsx` 点击 file 路径不变 (仍 setLastDiffPath + requestPopout('diff')),DiffPanel 自动接住

### 技术方案

#### 1. Main 端 IPC handler

走 git CLI(子进程)拿两边内容,**不**用 npm 包 `simple-git` 等(避免新增依赖)。

```ts
// project.gitFileDiff handler 伪码
async (input) => {
  const root = await projectStore.assertAllowed(input.projectRoot);
  const relPath = toPosixRelative(input.path, root);

  // 1. before = HEAD:relPath (git show)
  //    untracked file → before = '', isUntracked = true
  //    binary file → after read 直接 'isBinary: true' 短路, before/after 空
  const before = await tryGitShow(root, relPath);  // child_process spawn `git show HEAD:<relPath>`

  // 2. after = working tree content
  //    deleted file → after = ''
  const after = await fs.readFile(path.join(root, relPath), 'utf-8').catch(() => '');

  return { available: true, before, after, isUntracked: before === null, isBinary: ... };
}
```

安全:
- `projectStore.assertAllowed` 防 path traversal
- `path` 必须 inside projectRoot (复用 isPathInside)
- spawn `git` 用 `cwd: root` + 数组 args 不走 shell (防 injection)
- isBinary 检测: 读前 8KB 看是否含 NUL byte (跟 git 自己的 binary heuristic 一致)
- 大文件 cap: 工作树 file > 1 MB 直接返 `available: false` (Monaco diff render 这种没意义)

#### 2. Renderer DiffPanel 改造

```tsx
// DiffPanel.tsx fallback 逻辑伪码
useEffect(() => {
  if (!path || !projectRoot) return;
  // 优先 tool-call cache (现状)
  const r1 = await invoke('files.diff', { projectRoot, path });
  if (r1.ok && r1.data.available) {
    setDiff({ source: 'tool-call', before, after });
    return;
  }
  // fallback git working-tree diff (新)
  const r2 = await invoke('project.gitFileDiff', { projectRoot, path });
  if (r2.ok && r2.data.available) {
    setDiff({ source: 'git', before, after, isUntracked: r2.data.isUntracked });
  } else {
    setErr('No diff available');
  }
}, [path, projectRoot]);
```

DiffPanel 头部加 source pill 标记数据来源 ("Tool call" / "Working tree" / "Untracked"),让用户知道这是哪个语义的 diff。

### 接口契约

```ts
// packages/space-ipc-schema/src/channels/project.ts
export const projectGitFileDiffChannel = {
  name: 'project.gitFileDiff',
  direction: 'invoke',
  input: z.object({
    projectRoot: z.string().min(1).max(4096),
    path: z.string().min(1).max(4096),
  }),
  output: z.object({
    available: z.boolean(),
    before: z.string().max(MAX_DIFF_FILE_BYTES),  // 1 MB
    after: z.string().max(MAX_DIFF_FILE_BYTES),
    isUntracked: z.boolean().optional(),
    isBinary: z.boolean().optional(),
    reason: z.enum(['ok', 'file-too-large', 'is-binary', 'no-git', 'no-such-file', 'git-error']).optional(),
  }),
} as const;
```

`reason` 字段让 DiffPanel 显示精准错误 ("File too large for inline diff" / "Binary file" 等),不是泛 "miss"。

### 实现步骤

1. **schema**: 在 `packages/space-ipc-schema/src/channels/project.ts` 加 `projectGitFileDiffChannel` + export 到 index;rebuild schema 包
2. **main IPC handler**: `apps/desktop/electron/ipc/project.ts` 实现 `project.gitFileDiff`
   - 利用现有 `gitChanges` IPC 已经 spawn `git` 命令的模式,抽 helper `spawnGit(args, cwd) → Promise<{stdout, stderr, code}>`
   - 加 `isBinary` 检测 (NUL byte scan 前 8KB)
   - 加 1 MB 文件大小 cap
3. **renderer DiffPanel 改造**:
   - 加 `source: 'tool-call' | 'git' | null` state
   - 把现有 `files.diff` 调用包成 try-first,miss 时 fallback
   - 头部加 source pill UI
4. **测试**:
   - main 单测: 用 git fixture (`git init` + 写文件 + `git add/commit` + 改文件) 验证 modified / untracked / deleted / binary / large 五种 case
   - renderer 测试 fallback 链路 (mock 两 IPC: cache miss 时调 git diff)

### 验收标准

- [x] 历史 session 切到一个改过文件的 turn,点右侧 Changes 里某个文件 → DiffPanel 显示 working-tree diff (path 头 + Monaco 左右对比)
- [x] 同一文件如果 tool-call cache 有 (实时 session),仍优先显示 tool-call 那次的 before/after,头部 pill 标 "Tool call"
- [x] Untracked file (新增未 commit) → before 空,after 完整内容,pill 标 "Untracked"
- [x] Deleted file (从 working tree 删) → before HEAD 内容,after 空
- [x] Binary file → 拒绝 inline diff,显示 "Binary file — N bytes" 友好文案
- [x] 大文件 (> 1 MB) → 拒绝 inline diff,显示 "File too large for inline diff" 文案
- [x] 项目不在 git 仓库下 (`git rev-parse --git-dir` 失败) → 显示 "Not a git repository" 文案
- [x] 错误 / non-zero git exit → 显示 reason,不让 main throw 让 IPC envelope 走 HANDLER_ERROR
- [x] 性能: 单次 invoke `project.gitFileDiff` ≤ 200 ms (千行级 file),不阻塞 renderer

## Chore: 清理 `~/.kodax_space` 孤儿目录

### 背景

`~/.kodax_space/` 是早期版本 Space 残留(或来自其它工具的目录,具体来源待用户 `ls` 后确认)。当前 Space 代码 grep 整个 repo 完全不写它,但用户机器上仍然存在,造成两个 Space-related 目录共存的混乱感。

### 处理策略

`main.ts` 启动期加 best-effort 一次性清理钩子:

```ts
// apps/desktop/electron/main.ts (app.whenReady 内)
async function cleanupOrphanKodaxSpaceDir(): Promise<void> {
  const orphan = path.join(os.homedir(), '.kodax_space');
  try {
    const stat = await fs.stat(orphan).catch(() => null);
    if (!stat) return;
    // 安全: 只删 Electron userData 风的目录 (Cache / Local Storage / Preferences 等组合)
    // 如果是 KodaX SDK 数据风 (sessions / config.json) 则保留 + console.warn 让用户决定
    const entries = await fs.readdir(orphan).catch(() => []);
    const isElectronUserData = entries.some((e) =>
      ['Cache', 'Cookies', 'Local Storage', 'IndexedDB', 'GPUCache', 'Preferences'].includes(e),
    );
    if (isElectronUserData) {
      await fs.rm(orphan, { recursive: true, force: true });
      console.log('[startup] Cleaned orphan ~/.kodax_space (Electron userData residue)');
    } else {
      console.warn(`[startup] ~/.kodax_space exists but contains non-Electron data (${entries.length} entries); leaving alone.`);
    }
  } catch (err) {
    // best-effort,失败不影响启动
  }
}
```

只在确认是 Electron userData 风(组合特征)时才删,避免误删用户数据。其他情况只 console.warn 留痕。

不算 feature,不进 FEATURE_LIST.md;只在本 design doc 记录上下文 + 实现在 `main.ts` 一次性 commit。

### 验收标准

- [x] 机器上有 `~/.kodax_space` 且是 Electron userData 风 → 启动后被删,console 一行说明
- [x] 机器上有 `~/.kodax_space` 但是 KodaX SDK 数据风 → 保留 + console.warn
- [x] 机器上无该目录 → 启动期完全 no-op,不增加可观察启动延迟
- [x] 清理失败 (权限 / fs busy) → 失败,不阻塞 Space 启动
- [x] grep `'\.kodax_space'` 在 repo 里只出现在 main.ts 清理函数和本 design doc,其他地方零引用

## FEATURE_054: 视觉系统刷新 (refined dark+light token / Geist+JetBrains Mono / Lucide / 单一琥珀金强调)

### 需求概述

用户反馈"现在的视觉细节非常丑"。调研 Manus / Linear / Raycast / Vercel 后归纳出现代 AI 产品高级感的 6 条规律,并对照实际代码定位根因——**不是个别像素,是缺系统**:

1. **两套并行配色在打架** — [styles.css](../../apps/desktop/renderer/src/styles.css) 有一大坨 `html:not(.dark) .bg-zinc-900→...` 反转 hack(注释自承临时),组件混用字面 `zinc-*`(69 文件 / 1127 处)与语义类 → 必然不一致。
2. **配色彩虹,无品牌强调色** — amber/sky/emerald/blue/violet/purple/rose/orange/red 全在当强调色用。
3. **字号又小又挤** — 满屏 `text-[10px]/[11px]`。
4. **表面只 3 层纯灰** — 无浮层级、无暖调微染。
5. **emoji 当图标** — 📁 ● ✱ ☰ ◧ ◨ ⤢,跨平台渲染不一致。
6. **间距随意 / 边框硬线 / 无微交互 / 系统默认字体**。

已产出可点原型 [docs/design/visual-refresh-prototype.html](../design/visual-refresh-prototype.html) 并经用户多轮拍板(深+浅双主题、单一琥珀金、Geist+JetBrains Mono、Lucide、发送键 gradient+辉光)。本 feature 把原型那套 token 落进真实样式系统并分批迁移全 renderer。

> 设计规律(原型已验证):暗色优先 + 4 层表面靠亮度差分层 / 不用纯黑纯白(off-white `#ededef`、off-black `#1a1a1f`)/ 单一强调色只花在 hero CTA / 三级文字层次靠字号间距而非颜色 / base-4 间距 / 细节微交互。**浅色非反转**:白内容+灰 chrome、琥珀金 fill 保亮金质感(深色字垫底)、ink 取深档保证白底 AA。

### 影响范围

- **`styles.css`**:重写 token 块(`:root`=light / `html.dark`=dark,RGB-triplet 格式沿用现约定);收敛并最终删除 zinc 反转 hack。
- **`tailwind.config.cjs`**:扩 color token(新增 4 层表面 + accent 系 + 语义 + code/syntax)、`fontFamily.sans=Geist`、`boxShadow` 加 accent glow。
- **新增依赖**(Electron 无网 → 本地打包):`@fontsource/geist-sans`、`@fontsource/jetbrains-mono`、`lucide-react`。
- **renderer 全量**:69 文件 / 1127 处 `zinc-*` 分批迁语义类;emoji 全量换 Lucide;`text-[10/11px]` 上调到字号刻度。

> ⚠️ **零改动即时提升策略(降风险关键)**:**不重命名**现有 token(`--surface/--surface-2/--surface-3/--fg-*/--border-*/--hover-bg`)与 zinc 反转 map,只**重指它们的值**到新刻度 + **新增** `--bg-float`/`--accent*` 等。这样 69 个未迁移文件零改动即获新观感,迁移可分批、随时可发版,不存在"迁一半很丑"的中间态。
