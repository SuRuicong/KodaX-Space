## FEATURE_011: 内置终端（xterm.js + node-pty 单 tab）

### 需求概述

底部抽屉嵌入一个真实终端（不是模拟），与当前 active session 的 cwd 共享，用户可手动跑命令辅助开发。

### 影响范围

- 新建：`apps/desktop/renderer/features/terminal/TerminalPane.tsx`
- 新建：`apps/desktop/electron/ipc/terminal.ts`
- 依赖：`xterm`、`xterm-addon-fit`、`node-pty`

### 技术方案

- Main 用 `node-pty` 起一个 OS shell（Win: PowerShell；macOS/Linux: $SHELL）
- xterm.js 在 renderer 渲染，PTY 输出通过 IPC stream 转发
- 输入流：renderer keystroke → IPC → main → node-pty.write
- cwd 与 active session 的 projectRoot 同步（切 session 时 `cd` 过去）
- 大小调整：fit addon + IPC 通知 main pty.resize
- 关闭 Space 时 kill pty 进程

### 接口契约

```typescript
'terminal.create' → { cwd: string, cols: number, rows: number } → { ptyId: string }
'terminal.write'  → { ptyId: string, data: string } → void
'terminal.resize' → { ptyId: string, cols: number, rows: number } → void
'terminal.kill'   → { ptyId: string } → void

// stream events (main → renderer)
{ kind: 'terminal_output', ptyId, data: string }
{ kind: 'terminal_exit', ptyId, exitCode: number }
```

### 实现步骤

1. 装 node-pty + 平台 prebuilt 处理（Win 需要 Visual Studio Build Tools，CI 配好）
2. Main 终端管理器（单 pty）
3. xterm.js 集成 + fit addon
4. cwd 联动：切 session 时发 `cd <newCwd>\r` 给 pty
5. 主题适配（明 / 暗）
6. 复制粘贴快捷键
7. 关闭 cleanup

### 验收标准

- [ ] 跑 `npm install`、`git log`、`htop`（interactive）都正常
- [ ] cwd 与 active session 同步
- [ ] 关闭 Space 后无 pty 僵尸进程
- [ ] 100x100 输出无明显延迟（< 50ms）

## FEATURE_012: Subagent tree 视图

### 需求概述

当 KodaX 用 `dispatch_child_task` 起子 agent 时，右侧抽屉显示子 agent 树状结构 + 各自当前状态 + 可对单个子 agent 发指令或停止。

### 影响范围

- 新建：`apps/desktop/renderer/features/code/SubagentTree.tsx`
- 扩展：`session-event` schema 加 subagent 事件
- 依赖：KodaX 内核已有 `dispatch_child_task` / `send_message` / `task_stop` 三件套（FEATURE_120）

### 技术方案

- 监听 KodaX 的 child task 事件流，构建树（parent sessionId → child sessionId[]）
- 每个 child 节点显示：当前 tool / 模式 / Work 进度 / 状态（running/idle/done/failed）
- 右键菜单：`send_message`（推一条指令）/ `task_stop`（请求结束）
- 子 agent 完成后保留节点（淡色显示），点击可看完整 transcript

### 接口契约

```typescript
// session-event 扩展
{ kind: 'subagent_started', parentId, childId, task: string }
{ kind: 'subagent_status', childId, status: 'running'|'idle'|'done'|'failed', currentTool?: string }
{ kind: 'subagent_completed', childId, result: string }

// 反向 IPC
'subagent.sendMessage' → { childId, content } → { ok }
'subagent.stop'        → { childId } → { ok }
```

### 实现步骤

1. 探测 KodaX SDK 暴露的 child task 事件接口
2. SubagentTree 组件（react-arborist 或自实现）
3. 状态徽标 + 实时更新
4. 右键菜单
5. 点击节点切到 child session 完整视图（叠在主 conversation 上方或独立窗口）

### 验收标准

- [ ] 起 3 个并行 subagent 都显示
- [ ] Stop 某个 subagent 后状态变 "stopped"，KodaX 内核确实终止
- [ ] Send message 后子 agent 下一回合看到 coordinator instruction
- [ ] 子 agent 完成后节点保留可点击查看

## FEATURE_014: NAPI native-tokenizer

### 需求概述

第一个 Rust 加速器：用 `tiktoken-rs` 替换 `js-tiktoken`，把 token 计数提速 ~10×。详见 [ADR-002](../ADR/ADR-002-rust-integration-napi.md)。

### 影响范围

- 新建：`packages/native/tokenizer/`（独立 Rust + NAPI 项目）
- 修改：main 进程的 token 估算调用
- 发布：6 个 platform-specific npm packages

### 技术方案

- 用 napi-rs 模板：`yarn create napi-rs@latest`
- Rust crate 依赖：`tiktoken-rs = "0.5"`、`napi = "2"`、`napi-derive = "2"`
- 暴露 API：`encode(text: string, model: string) → number[]` + `count(text: string, model: string) → number`
- 编译矩阵（GitHub Actions）：win32-x64、win32-arm64、darwin-x64、darwin-arm64、linux-x64、linux-arm64
- npm 发布策略：metapackage `@kodax-ai/native-tokenizer` 用 `optionalDependencies` 引用平台包
- KodaX SDK 的 `estimateTokens()` 仍是 fallback；如果 NAPI 不可用（如未安装平台包）静默 fallback

### 接口契约

```typescript
// @kodax-ai/native-tokenizer
export function encode(text: string, model?: string): number[];
export function count(text: string, model?: string): number;
```

### 实现步骤

1. `yarn create napi-rs@latest @kodax-ai/native-tokenizer`
2. 实现 encode / count（tiktoken-rs 调用 BPE）
3. 6 平台 CI build matrix
4. 发到 npm（先 alpha tag）
5. Space main 引入：if NAPI 可用用 NAPI，否则用 KodaX SDK 的 estimateTokens
6. 性能 benchmark：vs js-tiktoken，验证 ≥ 5× 提速（大段 prompt）

### 验收标准

- [ ] 6 平台全部 build 通过
- [ ] 对 cl100k_base、o200k_base 等 tokenizer 与 OpenAI 官方 API 一致（采样 100 个 case）
- [ ] count("10k token text") 耗时 < 2ms（vs js-tiktoken ~20ms）
- [ ] NAPI 未装时 Space 不崩，silently fall back
- [ ] npm 上 `@kodax-ai/native-tokenizer` 包大小 metapackage < 5 KB，平台包 < 5 MB 每个

## FEATURE_030: AutoModeToolGuardrail bootstrap

### 需求概述

把 KodaX `@kodax-ai/coding` 暴露的 `bootstrapAutoMode` factory 接到 Desktop main，构造 `AutoModeToolGuardrail` 注入 `KodaXOptions.guardrails`。这是 FEATURE_029 的运行时配套——FEATURE_029 改 schema/UI，本 feature 真正让 auto mode 跑起来。

### 影响范围

- `apps/desktop/electron/kodax/real-session.ts` — 调用 bootstrapAutoMode；onEngineChange wire 到 emit
- `apps/desktop/electron/kodax/agents-md-loader.ts` — 新建：扫 projectRoot 与 ~/.kodax/AGENTS.md，返回 `AgentsFile[]`（FEATURE_034 跟进做完整版，本 feature 用最简实现）
- `apps/desktop/electron/kodax/kodax-sdk-types.d.ts` — 补 `bootstrapAutoMode` / `AutoModeBootstrapDeps` / `AgentsFile` / `Guardrail` 类型
- `apps/desktop/electron/test/auto-mode-bootstrap.test.ts` — 新建测试

### 技术方案

`bootstrapAutoMode` 是 async（要读 `~/.kodax/auto-rules.jsonc` 和项目级 `.kodax/auto-rules.jsonc`），所以在 `runRealStream` 开头 await 一次。Guardrail 是 lazy，只在第一次 tool call 时才真正构造，所以非 auto 模式的 session 零开销。

```ts
// runRealStream 中：
let guardrails: Guardrail[] | undefined;
if (this.permissionMode === 'auto') {
  const result = await bootstrapAutoMode({...});
  guardrails = [result.getGuardrail()];
  // result.rulesLoadResult 可 emit 给 renderer 作为 status_bar 提示
  this.emit({
    kind: 'auto_rules_loaded',
    sessionId: sid,
    sources: result.rulesLoadResult.sources,
    errors: result.rulesLoadResult.errors,
  });
}
options.guardrails = guardrails;
```

### 接口契约

```ts
// 新增 SessionEvent
{ kind: 'auto_rules_loaded', sessionId, sources: string[], errors: string[] }
```

### 实现步骤

1. SDK types 扩 `bootstrapAutoMode` + 依赖类型
2. `agents-md-loader.ts` 最简实现：读 `${projectRoot}/AGENTS.md` 和 `~/.kodax/AGENTS.md`，返回 `[{path, content}, ...]`
3. real-session.ts 加 `maybeBootstrapAutoMode` 私有方法
4. emit auto_rules_loaded event
5. 测试：mock bootstrapAutoMode、验证 mode='auto' 时 guardrails 注入

### 验收标准

- [ ] auto mode session 启动后，~/.kodax/auto-rules.jsonc（手动放）能被 loadAutoRules 读到，emit auto_rules_loaded 反映 sources
- [ ] auto mode + engine llm 跑一个会触发 classifier 的 prompt，KodaX log 能看到 sideQuery 调用
- [ ] denial threshold 触发后 onEngineChange callback 被调用，emit auto_engine_change

## FEATURE_033: Sessions tree + fork + rewind

### 需求概述

KodaX `~/.kodax/sessions/` 已持久化 session 历史 + parent/child 关系。Desktop 应当：
- 显示 session 列表（含分组：当前项目 / 其他项目 / 已 archived）
- 支持 fork：从某个历史 turn 起 branch 出新 session
- 支持 rewind：把当前 session 回退到某个 turn，删除其后所有 turn

### 影响范围

- `packages/space-ipc-schema/src/channels/session.ts` — 新增 channels：`session.list`（扩展现有）、`session.fork`、`session.rewind`
- `apps/desktop/electron/kodax/session-store.ts` — 新建：读 ~/.kodax/sessions/ 解析 KodaX session 持久化文件
- `apps/desktop/electron/kodax/host.ts` — fork / rewind handler
- `apps/desktop/renderer/src/shell/SessionTreePanel.tsx` — 新建：左侧 sidebar Recents 区域升级
- `apps/desktop/renderer/src/shell/SessionMenu.tsx` — 把现有 Fork / Rewind 占位项接通

### 技术方案

KodaX session 文件格式：`~/.kodax/sessions/<sessionId>.json`，含 `turns[]` + `parentSessionId?`。Desktop session-store 加载时构建 in-memory tree。

fork：复制 turns[0..forkTurnIdx]，新 sessionId，写 parentSessionId 指向原 session。
rewind：truncate turns[rewindTurnIdx+1..]，原文件就地改写（带 atomic write）。

### 验收标准

- [ ] sidebar Recents 显示当前项目所有 session + 分组（active / archived）
- [ ] Session menu Fork 弹 modal 选 turn 索引，确认后新 session 创建
- [ ] Rewind 确认对话框（不可逆）→ 当前 session turn 截断
- [ ] tree 视图能看到 parent → child 关系（缩进显示）
