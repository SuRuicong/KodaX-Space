# v0.1.15 — Workflow Harness 支持（多 agent 编排接通 GUI）

> 起草 2026-06-17。对标 KodaX SDK **FEATURE_217**（0.7.49 动态工作流引擎）+ **FEATURE_229**（0.7.50 工作流进程事件 / SDK 进度面），把 KodaX 的多 agent 编排能力在 Space 的图形界面里接通。

## 版本目标

KodaX 0.7.49/0.7.50 把「agent 生成并运行一段编排脚本、fan-out 多个子 agent 并行干活」做成了可订阅的一等进程。REPL 在终端 inline/fullscreen 渲染它；本版本给 Space 做**等价的图形面**——进度树、run 控制、库入口、子 agent 遥测、结果/artifact 展示。

**核心原则**：Space **零编排逻辑**，只订阅 SDK 的 `WorkflowProcessSnapshot` 流 → 转发 → 渲染 → 回传控制。编排 / 生成 / worktree / token 预算 / 子 agent 权限闸全归 KodaX 内核。

依赖：F060 是地基先做；F061/F062/F065/F066 建于其上；F063/F064 是「人怎么发起工作流」的两个入口。

## 关键决策（2026-06-17）

1. **只给 Coder surface**，Partner 暂不开。
2. **artifact 走方案 A**：workflow 产物桥进现有 `artifactStore`，与 agent 直接产的统一面板，复用 F057-F059 预览/版本/导出/弹窗。
3. **run→session 归属是 SDK 缺口**（`WorkflowProcessSnapshot`/`processMetadata` 无 host 自定义槽位）——已开需求转交 KodaX；补齐前 Space 用自持久化的 `runId→{sessionId,surface}` 映射做 interim。
