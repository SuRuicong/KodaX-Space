## Feature_107 KodaX 0.7.54 SDK Catch-up: Learning, Recovery, Extensions

### 1. Goal

`0.7.54` is not only a dependency bump. It adds a new procedural-learning review loop, session recovery seeds, filesystem extension discovery/runtime composition, GLM provider-model refresh, and manual self-knowledge updates. Space should expose the user-visible parts quickly, while keeping destructive or cross-runtime changes behind reviewable gates.

This feature turns the former `v0.1.25` patch reserve into the SDK catch-up lane before F103 starts at `v0.1.26`.

### 3. Implementation Phases

Completed SDK parity surface:

- Upgrade Space to `@kodax-ai/kodax@0.7.54`.
- Align GLM provider defaults and renderer model context caps.
- Add slash learning inbox commands using SDK store APIs.
- Add skill usage/trust ledger diagnostics via `/learn ledger` and `/skill ledger`.
- Add recovery diagnostics and seed preview via `/recover candidate|prompt|seed`.
- Add SDK filesystem-extension discovery via `/extensions sdk` and explicit runtime activation via `/extensions sdk load`.
- Inject the SDK extension runtime into real sessions only when `KODAX_SPACE_ENABLE_SDK_EXTENSIONS=1` is set.
- Add Space manual coverage for extensions.
- Wire completed-turn learning recording after successful turns while keeping proposal creation candidate-gated.

Future rich UI enhancements:

- Add a Learning Inbox panel with proposal counts, kind filters, apply-plan diff preview, and approval/rejection controls.
- Add skill usage/trust ledger views to the Skills surface.
- Add a first-class recovery-session fork action once Space exposes a persisted seed-to-session writer.

### 4. Acceptance Criteria

- `/learn pending` lists SDK learning proposals for the current project.
- `/learn diff <id>` shows proposal detail and any safe apply plan without applying it.
- `/learn reject <id> [reason]` updates the SDK proposal store.
- `/learn approve <id>` delegates to SDK safety gates and reports blocked consumer impact clearly.
- `/skill pending`, `/workflow pending`, and `/memory pending` show filtered proposal queues.
- GLM defaults and context hints match SDK `0.7.54` model lineup.
- Future recovery and extension work is tracked here rather than hidden as ad hoc SDK drift.
