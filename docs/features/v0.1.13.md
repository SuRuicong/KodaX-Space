## FEATURE_050: Space 有头浏览器能力（in-process registerTool，双 surface）

### 需求概述

有头浏览器需要显示面，KodaX 内核（headless、要跑 CLI/CI/Linux server）结构上托不了；Space 是 Electron、自带 Chromium，**把有头浏览器实现成 in-process 自有工具，经 `extensionRuntime.registerTool` 注册进 runtime，agent 当原生工具调用——不走 MCP、不劳内核**（[ADR-007](../ADR/ADR-007-partner-surface-model.md)）。能力：导航、JS 渲染、点击/填表交互、截图、结构化抽取、带出处。服务**双 surface**：Coder（操控本地 dev server、e2e、截图验证），Partner（研究取证）。

### 影响范围

- 新 `apps/desktop/electron/browser/`：
  - `BrowserEngine.ts`——基于 Electron `BrowserWindow`/`BrowserView`（off-screen 或可显示）封装：`navigate / click / fill / screenshot / extract / readText`
  - `browser-tool.ts`——把上述能力包成一个或一组 `LocalToolDefinition`，经 `registerTool` 注册
- **kodax 集成**：起 session 时把 browser tool 注册进 `extensionRuntime`（Coder 全集 + Partner 白名单都纳入，工具标 `network`）。
- **oversight**：浏览器外呼/导航走现有 permission 模型（按 session 粒度可关；导航到新域名可二次确认）。
- **headful UI**：一个可显示的浏览器视图（用户能看 agent 在干什么），可停靠右栏/独立窗口；默认对 Partner 研究场景显示。

### 技术方案

```ts
// electron/browser/browser-tool.ts —— 经 registerTool 注册，全 harness 下可用（依赖 SDK R1 确认点）
const browserTool: LocalToolDefinition = {
  name: 'browser',
  description: 'Navigate/interact/extract from web pages via a real headful Chromium.',
  parameters: { action: 'navigate'|'click'|'fill'|'screenshot'|'extract'|'read', ... },
  capabilities: ['network'],                 // 供 R3 维度白名单识别
  handler: async (input, ctx) => engine.dispatch(input, ctx),
};
extensionRuntime.registerTool(browserTool);
```

- **后端**：Electron `BrowserView` 复用主进程 Chromium，无需 bundle Playwright（省体积）；若需更强自动化语义，留可替换后端接口（fallback：本地 Playwright/CDP）。
- **headful**：默认开一个可见 view（oversight）；可配置 off-screen（纯抽取场景）。
- **抽取**：`extract` 跑页面内 JS 取结构化数据 + 留 URL/选择器出处（faithfulness）。
- **降级**：session teleport 到 CLI（无显示面）→ 浏览器工具不可用，自动降级 `toolWebFetch`（headless 轻量）。
- **R1 确认点**：embedder 经 `registerTool` 注册的工具能否在 `runManagedTask` 全 harness 下被 agent 调用 + 受 permission 门控——**未通过则 fallback 本地 MCP server 挂载**（[ADR-007](../ADR/ADR-007-partner-surface-model.md) 依赖段）。design 标注此实测分叉。

### 接口契约

- 工具经 `registerTool` 注册，非 IPC channel。
- headful view 控制走少量 IPC：`browser.show/hide/dock`（renderer ↔ main）。
- permission：复用现有 permission 弹窗，tool=`browser`、risk 按 action（navigate 新域名 = 提示）。

### 实现步骤

1. `electron/browser/BrowserEngine.ts`：BrowserView 封装 navigate/interact/screenshot/extract。
2. `browser-tool.ts`：`LocalToolDefinition` + `registerTool`；标 `network` 能力。
3. **实测 R1 确认点**：注册的工具在 managed run 全 harness 下可用？通过→直用；否则→本地 MCP fallback（记录结论到 [[kodax_sdk_export_gaps]]）。
4. headful view UI（可见/停靠）+ `browser.show/hide/dock` IPC。
5. oversight：browser 动作接 permission；新域名导航二次确认。
6. CLI 降级路径：无显示面时不注册 browser，留 `toolWebFetch`。
7. 测试：navigate + extract 一个 JS 渲染页面拿到结构化数据 + 出处；权限门控生效；Coder 也能调（截图本地 dev server）。

### 验收标准

- [ ] agent 在 Partner 调 `browser` 导航到一个 JS 渲染页面并抽取结构化数据，带 URL 出处。
- [ ] 有头视图可见，用户能看到 agent 的导航/点击过程（oversight）。
- [ ] 同一 browser 工具在 Coder 可用（截图本地 dev server / 打开页面验证）。
- [ ] 浏览器动作受 permission 门控；导航新域名二次确认；可按 session 关闭网络。
- [ ] R1 确认点结论落到 design + 记忆：直用 registerTool 还是 MCP fallback。
- [ ] teleport 到 CLI 场景自动降级 `toolWebFetch`，不报错。
- [ ] 不 bundle 额外浏览器二进制（复用 Electron Chromium）—— 安装包体积零显著增长。

## FEATURE_051: Partner skill pack：文档事务（PDF 抽表 / 总结 / slides 生成）

### 需求概述

第二组 Partner skill——纯文档事务：从 PDF/图片抽表、长文档/文件夹总结、生成 slides。复用 Space 已有的 office 能力（docx/pdf/pptx/xlsx skill + 图像理解）+ F024 富预览，产出走 F048 artifact。

### 影响范围

- 新 `electron/skill/partner-packs/doc-ops/`：
  - `extract-table-from-pdf`——PDF/图片 → 结构化表格（xlsx/csv artifact）
  - `summarize-folder`——一个目录的多文档 → 总结报告 artifact
  - `generate-slides`——要点/文档 → pptx/slides artifact（复用 pptx 能力）
- 复用：现有 office skill / 图像理解（截图 OCR）；web 类源经 F050 浏览器。

### 技术方案

每个 skill 在 Partner 白名单（F047）内运行，输入来自 Sources（F046 左栏）或拖入文件，输出 `artifact.write`（F048）。抽表优先本地 PDF 解析；扫描件/图片走图像理解 OCR。slides 生成复用 pptx 能力，产出 pptx artifact 经 F024 预览。

### 接口契约

无新 IPC；skill 经现有通道，产出经 F048。

### 实现步骤

1. 建 `doc-ops/` + 3 个 skill 定义。
2. 接 office 能力（pdf 抽取 / xlsx 写 / pptx 生成）+ 图像 OCR。
3. 产出 → `artifact.write`，kind 对应 xlsx/pptx/report。
4. Sources 拖入文件 → skill 输入打通。
5. 测试：一个含表格的 PDF → 抽出 xlsx artifact，行列正确；一个目录 → 总结 artifact 覆盖各文件。

### 验收标准

- [ ] 含表格 PDF → `extract-table-from-pdf` 产出 xlsx/csv artifact，数据结构正确。
- [ ] 扫描件/图片表格 → 经 OCR 抽取（图像理解能力）。
- [ ] 一个文档目录 → `summarize-folder` 产出覆盖各文件的总结报告 artifact。
- [ ] `generate-slides` 产出 pptx artifact，经 F024 预览。
- [ ] 全部产出登记 F048，可预览/迭代/导出。
- [ ] 全程 Partner 白名单内（无 bash）。
