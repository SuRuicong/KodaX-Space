# v0.1.22 - Provider / Queue Patch

Release date: 2026-06-22

## Scope

v0.1.22 is a patch-only release. It does not assign new planned feature IDs; it consumes the reserved patch lane for trusted internal custom providers, queue correctness, ask_user bridge coverage, and release metadata alignment.

## Fixed

- Custom provider Settings flow can explicitly skip URL safety validation for trusted internal HTTP/IP gateways, while the default path still enforces HTTPS and dangerous-scheme blocking.
- KodaX config custom providers retain the trusted path, preserving existing direct-config internal provider behavior.
- Space-owned per-session follow-up queues replace user-prompt use of the SDK main-thread queue, preventing another running Space session from draining the wrong prompt.
- Resumed sessions avoid stale provider/model pairings by only reusing configured models that belong to the selected provider.
- The title sanitizer regex no longer embeds an actual NUL byte in source.
- SDK ask_user question/select/input prompts are bridged to the Space modal path.
- MCP manager init/reload/dispose lifecycle races are hardened.

## Release Alignment

- Root, desktop, IPC schema, UI kit, and lockfile versions are aligned to `0.1.22`.
- Runtime capability contract is `space-v0.1.22`.
- CHANGELOG, usage guide, capability ledger, and feature matrix now describe v0.1.22 as the current patch release.
- v0.1.23 and v0.1.25 remain patch lanes; v0.1.24 remains F105 customer timebox entitlement MVP.
