# v0.1.7 — MCP management follow-up

> 与 KodaX SDK 公开 MCP manager API 同步发布。本版本只有一个 feature：把 F036
> (MCP popout) 从 read-only 升级到完整的 start/stop/log/tool catalog 管理。
>
> 触发条件：KodaX SDK 发布的版本含公开 MCP manager API (启停 server、读 stdout/stderr、
> 拉 tool catalog、热重载 config)。

## 版本目标

F036 在 v0.1.1 alpha.1 作为 **read-only listing** 落地（直接读 `~/.kodax/config.json`
`mcpServers` 字段）——用户能看到已配置 server，但不能控制生命周期。v0.1.7 接 SDK
MCP manager API 后补齐完整能力，与 REPL `/mcp` 命令对等。

## FEATURE_039: F036 MCP 管理完整版

### 需求概述

让 desktop 的 MCP popout 走 KodaX SDK 公开 MCP manager API，能启停 server、看 log、
看 tool catalog。F036 已经搭好的 popout UI + IPC 框架不动，只把 main 端 config-reader
换成 SDK 调用 + 加几个 mutation IPC（mcp.start / mcp.stop / mcp.refresh / mcp.logs）。

### 影响范围

- `apps/desktop/electron/mcp/config-reader.ts` — 内部实现替换为 SDK 调用
- `apps/desktop/electron/mcp/manager.ts` (新建) — 包装 SDK MCP manager，加 Space 端缓存 + log 转 IPC push
- `apps/desktop/electron/ipc/mcp.ts` — 加 mcp.start / mcp.stop / mcp.refresh / mcp.logs 通道
- `packages/space-ipc-schema/src/channels/mcp.ts` — 扩展 server meta（status / lastStartedAt / toolCount）+ 新增 mutation channels
- `apps/desktop/renderer/src/shell/popouts/McpPanel.tsx` — 加启停按钮、状态颜色、log 视图、tool catalog 弹窗

### 前置依赖

- KodaX SDK 发布 MCP manager 公开 API，含：
  - `listMcpServers({ projectRoot })` → 含 status (running/stopped/errored)、lastStartedAt、配置等
  - `startMcpServer(name)` / `stopMcpServer(name)`
  - `getMcpServerLogs(name, { tail? })` 或 push stream
  - `getMcpServerTools(name)` → 已 discover 的 tool definitions
  - 可选 `watchMcpServers()` 文件 / 状态变化推送

### 技术方案

复用 F036 已搭框架，把 main 端 `discoverMcpServers` 内部从"自己 parse config.json"
换成"SDK listMcpServers"。renderer popout UI 已经为 status 列预留 (transport label 旁边
加一个 status dot)。

mutation channels 走 main → SDK 转发；result 走 IPC envelope。logs 走 push 频道
(类似 session.event)，让 UI 滚动显示。

### 验收标准

- [ ] MCP popout 显示每个 server 的实时 status (running / stopped / errored)
- [ ] 点击 server 启停按钮 → status 变化 + tool catalog 更新
- [ ] 日志 tab 显示 server stdout/stderr 滚动
- [ ] tool catalog tab 显示该 server 暴露的所有 tool（名字 + 描述 + JSON schema）
- [ ] config.json 外部修改后 popout 自动刷新（或刷新按钮）
- [ ] /mcp 命令在 / popover 也能调起（refresh / status，对齐 REPL）

### 实现步骤

1. 升 `@kodax-ai/kodax` 到含 MCP manager API 的版本
2. 实现 `apps/desktop/electron/mcp/manager.ts` 包 SDK
3. 替换 `config-reader.ts` 内部实现（保持 discoverMcpServers 外部 API 不变）
4. 加 mutation IPC + push 频道
5. 升级 McpPanel UI：状态显示 + 启停按钮 + log/tool tabs
6. 测试：mock SDK manager → 验证 IPC / 状态流转

### 风险

- **SDK API shape 未定型**：v0.1.1 实现只 expect`listMcpServers` 之类接口，但 SDK
  实际暴露形态由 KodaX 团队定。届时 manager.ts 适配
- **MCP server stdio 隐私**：log 包含 stdout/stderr 原文，可能含敏感 path / env hints；
  Space 不做内容 redaction（SDK 自己 strip secret env value），但 log 上限要严格
  (max bytes / time window)
- **跨进程并发**：REPL 同时跑时 server 状态可能由两边都触发；start/stop 需要 SDK 层
  做锁（同 F038 rewindSession 'session_busy' 概念）

### 不做的事

- 不允许 Space 直接写 config.json 加/删 server——保持只读编辑路径（用户改 config，
  Space 反映）。v0.1.8+ 可加 GUI 编辑器
- 不为 server health check 加 ping/probe 机制——SDK manager 自己负责
