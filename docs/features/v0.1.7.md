## FEATURE_011: 真 PTY 单 tab 终端

### 需求概述

替换原 popout 的"bash 工具历史 viewer"为真 PTY shell。F023 多 tab 的前置。

### 实现要点

- `apps/desktop/electron/terminal/ptyHost.ts` — main 端 PtyHost 单例，Map<uuid, IPty>
- 4 IPC channel：`terminal.create` / `.write` / `.resize` / `.kill` + push `.output` / `.exit`
- 服务端定 shell（cmd.exe / $SHELL），renderer 不能注入 arg
- ENV allowlist（PATH/HOME/USER/TERM/LANG/Win 必备项），剥所有 `*_KEY` `*_TOKEN`
- assertAllowed + fs.realpath 双层 symlink-safe
- 退出时 PTY 强制清理，防孤儿
- 8 单测 spawn 真 shell 验证生命周期

### 验收

- [x] Win11 cmd.exe + Mac zsh + Linux bash 都能起
- [x] xterm.write/onData 双向通畅
- [x] PTY 退出后 popout UI 显示 `[exit N]`
- [x] before-quit 时 disposeAll 强杀，无 zombie

## FEATURE_023: 内置终端多 tab

### 需求概述

PtyHost 本来就支持多实例，纯 renderer 工作：tab bar + activeId state，每个 tab
独立 Terminal 实例。

### 实现要点

- `apps/desktop/renderer/src/features/terminal/TerminalManager.tsx` — 单 useReducer
- `tabsReducer.ts` 纯 reducer（exported for test），types 抽出
- 非 active tab 用 `display:none` 隐藏，PTY 保活
- Terminal.tsx ResizeObserver 加 0×0 guard，防 hidden tab 收到 1×1 SIGWINCH
- MAX_TABS=10（UI cap + main 端 IPC 硬上限双层）
- 关闭最后一个 tab 自动开新 tab；关 popout 用顶栏 ×
- 12 reducer 单测覆盖 ADD/CLOSE/ACTIVATE 不变量

### 不在本 feature

- 键盘快捷键（Ctrl+T/Ctrl+W 会撞 BottomBar / xterm）— 后续可加 popout-scoped
- tab 状态持久化跨 session — MVP 不做

## FEATURE_024: 文件富预览（PDF / docx / xlsx）

### 需求概述

Preview popout 路由：text 走 Monaco，PDF/docx/xlsx 走对应 lazy viewer。

### 实现要点

- 新 IPC `files.readBinary`：assertAllowed + resolveInsideProject + maxBytes 兜底
- `binaryUtils.ts` — base64ToBytes + detectKind + PREVIEW_SIZE_CAPS
- 3 个 lazy viewer 各自一个 chunk（main bundle 4.15MB 不变）：
  - PdfViewer：pdfjs-dist 4.10 ESM；isEvalSupported:false + disableAutoFetch:true 硬化
  - DocxViewer：mammoth + 自写 DOMParser allowlist sanitizer（tags + attrs + href scheme）
  - XlsxViewer：SheetJS (CDN tarball) → sheet_to_json → React 渲染 table（无 innerHTML）
- 大小上限：PDF 50MB / docx 10MB / xlsx 10MB；超限 truncated=true 空 base64

### 验收

- [x] 真 PDF（多页 + 嵌入字体）正常翻页
- [x] 真 docx 表格 + 标题 + 列表正确渲染
- [x] 真 xlsx 多 sheet tab 切换正常
- [x] 超限文件给清晰错误提示

## FEATURE_026: ⌘Shift+P 命令面板

### 需求概述

VS Code 同款的 "search anything → execute" 入口。4 个分组：Actions / Sessions / Files / Slash。

### 实现要点

- `apps/desktop/renderer/src/lib/fuzzy.ts` — JS fzf-lite scorer，FuzzyMatcher 抽象
- 多起点 scan 避开 greedy-first-match；11 单测
- `CommandPalette.tsx` — modal + 分组列表 + 键盘 nav
- 改键：原 plan ⌘K 与 F018 Quick Ask 冲突，让位 ⌘Shift+P
- 模块私有 inputBridge 替 window CustomEvent（消除 ambient inject capability）
- FIFO bucket map 解决 searchText collision 重复项被遮蔽
