## Part A · 批次叙事

### 背景与动机

KodaX-Space 在 **F054（视觉系统刷新）** 与 **F060（视觉质量三档：液态玻璃 / 景深 / 极光）** 之后，已经建立了一套**静态视觉语言**——单一琥珀金强调、refined dark+light token、Geist + JetBrains Mono、液态玻璃面板、分层柔影、克制的环境极光。

但「**话术系统**」——用户停留时间最长、承载核心信息流的**对话流**（`ConversationStreamV2` + `bubbles` 组件族）——至今仍是**最「静」的表面**：

- 消息出现是瞬切（无 fade-up、无 stagger）
- 工具组 `ToolCluster` 展开 / 折叠是瞬切（无高度动画）
- `ThinkingBlock` 折叠是瞬切
- `ToolCallCard` running→done 状态切换是瞬切（无颜色过渡 / 完成高光）
- 时间线 `TimelineMarker` 直接出现（无 pop）
- 流式 `ActivitySpinner` 已有 Braille 帧循环，但回复气泡本身的「正在打字」光标缺失
- 复制成功只在文字上切「copied」（无视觉反馈脉冲）

这与已建立的**动态玻璃/景深视觉语言**形成割裂：chrome 在动（极光漂移、光标高光、按钮浮起），而**内容层完全静止**。用户感知是「壳子很精致，但内容像 PPT 截图」。

参照 huashu-design `references/animation-best-practices.md` 提炼的「Anthropic 级」动画哲学——**动画是物理学的三层展开（叙事节奏 / 运动曲线 / 表达语言）**，`expoOut` 给数字元素「物理重量感」——本批次把这套运动语言下沉到话术系统的**每一个状态切换**，让对话流从「截图」变成「可伸手进去的空间」。

### 设计目标

1. **全量覆盖**：覆盖话术系统所有可观测的状态切换（见 Part B §动画清单），不留交互盲点。
2. **现代化 / 美观**：遵循 huashu-design 动画语法——`expoOut` 主 easing、`overshoot` 弹性、30ms stagger、Anticipation→Action→Follow-through、关键结果前克制（礼让观众）。
3. **易改 / 易维护**：单一配置源（`motion.ts` 动画参数 token + CSS 变量），组件化（`Reveal` / `Collapse` 可复用包装），零新重依赖。
4. **可配置**：动画时长 / easing / stagger 间距全部走 token，改一处全局生效；遵循现有视觉质量三档门控。
5. **无障碍 / 性能**：尊重 `prefers-reduced-motion`；尊重 `q-minimal` 档（极简档退化为瞬切）；只动 transform/opacity（GPU 合成，不触发 layout），对话滚动区绝不挂 `backdrop-filter`（沿用 F060 性能护栏）。

### 技术选型（关键决策）

| 选项                                     | 结论                                      | 理由                                                                                                                                                                                              |
| ---------------------------------------- | ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **framer-motion / motion**               | ❌ 不引入                                 | 项目哲学「极简且智能」一贯避免重依赖（F044 明确拒绝 simple-git）。话术动画需求（fade-up / height-collapse / 状态 crossfade）纯 CSS keyframes + 一个 thin hook 完全覆盖，新增 ~40KB 运行时不划算。 |
| **CSS-first + thin React 层**            | ✅ 采纳                                   | 与现有 `.ix-zone` / `.glass` / `.lift` 同源（都在 `styles.css`）；零运行时开销；`prefers-reduced-motion` 与 `q-minimal` 门控沿用既有模式；改 token 即全局生效。                                   |
| **Web Animations API (element.animate)** | ⚠️ 仅用于一次性「完成高光」等 JS 触发场景 | 大部分用 CSS；个别需要「事件触发、播放一次即止」的（如 tool done 的 flash）用 WAAPI，避免常驻 keyframe。                                                                                          |

**结论**：CSS-first，新增一个 `motion.ts`（纯配置，导出 duration/easing 常量 + CSS 变量名 + 一个 `usePrefersReducedMotion` hook），新增两个可复用组件 `Reveal` / `Collapse`，动画 keyframe 与 token 挂进现有 `styles.css`。

### 可维护性策略

1. **单一配置源**：所有时长 / easing / stagger 集中在 `motion.ts` 导出常量，并在 `styles.css` `:root` 里镜像成 CSS 变量（`--motion-*`、`--ease-expo` 等）。改动画全局节奏 = 改一个文件。
2. **组件化**：`Reveal`（入场 fade-up + stagger）与 `Collapse`（高度折叠）是通用包装，对话流 / 工具组 / 思考块 / 系统通知统一复用，不在各组件里重复写动画。
3. **门控复用**：完全沿用 F060 的 `html.q-minimal` 档位 + `prefers-reduced-motion` 既有模式，新增 keyframe 一律 `html:not(.q-minimal)` 前缀 + `@media (prefers-reduced-motion)` 兜底。极简档 / 减弱动效用户 = 瞬切，零退化。
4. **零新依赖**：纯 CSS + 两个极小 React 组件 + 一个 hook。无打包体积变化。
5. **文档化「为什么」**：每个 keyframe 在 `styles.css` 注释里写明触发时机与运动意图（对齐现有 `.ix-zone` 的注释风格），未来调参有人可追溯。

## Part B · FEATURE_068 设计块

## FEATURE_068: 对话流全量交互动画系统（Conversation Motion System）

### 1. 需求概述

为话术系统（对话流 `ConversationStreamV2` + `bubbles` 组件族）的**每一个状态切换**加上现代化、美观、物理感强的动画，遵循 huashu-design 动画语法。要求易改、易维护、参数可配置、组件化，复用现有视觉质量三档门控与无障碍护栏，零新重依赖。

**一句话**：让对话流从「PPT 截图」变成「有重量、可伸手进去的空间」，且改一处 token 全局调参。

### 3. 技术方案

#### 3.1 动画参数配置（`motion.ts`）—— 单一改动源

```ts
// apps/desktop/renderer/src/lib/motion.ts
// 所有动画节奏集中于此。改这里 = 全局调参。styles.css :root 镜像同名 CSS 变量。

/** 主 easing（入场 / 面板 / focus）—— expoOut：迅速启动缓慢刹车，给数字元素物理重量感。
 *  huashu-design animation-best-practices §2：cubic-bezier(0.16, 1, 0.3, 1) */
export const EASE_EXPO = 'cubic-bezier(0.16, 1, 0.3, 1)';
/** 弹性 easing（toggle / 按钮弹出 / 强调交互）—— overshoot。
 *  cubic-bezier(0.34, 1.56, 0.64, 1) */
export const EASE_OVERSHOOT = 'cubic-bezier(0.34, 1.56, 0.64, 1)';
/** 物理 easing（几何体归位 / 自然落位）—— spring 近似。
 *  cubic-bezier(0.34, 1.3, 0.64, 1)（弱化 overshoot 避免文字抖） */
export const EASE_SPRING = 'cubic-bezier(0.34, 1.3, 0.64, 1)';

/** 时长档（ms）—— 三档覆盖全场景，避免每个动画各写一个魔数。 */
export const DURATION = {
  fast: 150, // 微反馈：caret 旋转、copy 脉冲、ring 出现
  normal: 260, // 主力：消息入场、marker pop、状态 crossfade
  slow: 380, // 大面板：cluster / thinking 折叠展开
} as const;

/** stagger 间距（ms）—— 列表入场相邻元素延迟。
 *  huashu-design §4.3：30ms stagger；对话流稍密用 24ms 避免长列表拖尾。 */
export const STAGGER_MS = 24;
/** stagger 封顶数 —— 超过 N 条后不再累加延迟（避免 50 条历史回填时最后一条等 1.2s）。 */
export const STAGGER_CAP = 8;

/** 这些常量在 styles.css :root 里镜像成 CSS 变量，keyframe / 工具类引用变量而非魔数。 */
export const MOTION_CSS_VARS = {
  easeExpo: '--ease-expo',
  easeOvershoot: '--ease-overshoot',
  easeSpring: '--ease-spring',
  durationFast: '--motion-fast',
  durationNormal: '--motion-normal',
  durationSlow: '--motion-slow',
} as const;
```

#### 3.2 CSS token + keyframe（`styles.css`）

```css
:root {
  /* 镜像 motion.ts 常量（单一改动源 → CSS 引用变量） */
  --motion-fast: 150ms;
  --motion-normal: 260ms;
  --motion-slow: 380ms;
  --ease-expo: cubic-bezier(0.16, 1, 0.3, 1);
  --ease-overshoot: cubic-bezier(0.34, 1.56, 0.64, 1);
  --ease-spring: cubic-bezier(0.34, 1.3, 0.64, 1);
}

/* —— 消息入场：fade-up，expoOut。mount 即播一次， forwards 锁定终态。 —— */
@keyframes ks-msg-in {
  from {
    opacity: 0;
    transform: translateY(8px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
/* —— 时间线 marker pop：scale overshoot（弹性入场，呼应 huashu §4.6 Anticipation→Action→Follow-through）—— */
@keyframes ks-marker-pop {
  0% {
    opacity: 0;
    transform: scale(0.4);
  }
  60% {
    opacity: 1;
    transform: scale(1.15);
  } /* overshoot */
  100% {
    opacity: 1;
    transform: scale(1);
  } /* settle */
}
/* —— 工具完成高光：一次性 flash（WAAPI 触发更可控，keyframe 作降级）—— */
@keyframes ks-done-flash {
  0% {
    box-shadow: 0 0 0 0 rgb(var(--ok) / 0.5);
  }
  100% {
    box-shadow: 0 0 0 8px rgb(var(--ok) / 0);
  }
}
/* —— 流式打字光标：竖条呼吸（替代纯文字省略号，物理感更强）—— */
@keyframes ks-caret-blink {
  0%,
  100% {
    opacity: 1;
  }
  50% {
    opacity: 0;
  }
}
/* —— 复制成功脉冲：scale 弹一下 —— */
@keyframes ks-copy-pulse {
  0% {
    transform: scale(1);
  }
  40% {
    transform: scale(1.18);
  }
  100% {
    transform: scale(1);
  }
}
/* —— 系统通知入场：从顶部滑入 + 淡入 —— */
@keyframes ks-notice-in {
  from {
    opacity: 0;
    transform: translateY(-6px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* 工具类：全部 html:not(.q-minimal) 前缀（极简档瞬切）+ reduced-motion 兜底 */
html:not(.q-minimal) .reveal {
  animation: ks-msg-in var(--motion-normal) var(--ease-expo) both;
}
html:not(.q-minimal) .reveal-marker {
  animation: ks-marker-pop var(--motion-normal) var(--ease-overshoot) both;
}
/* …其余工具类同构… */
@media (prefers-reduced-motion: reduce) {
  .reveal,
  .reveal-marker {
    animation: none;
  }
}
```

#### 3.3 通用组件（`Reveal` / `Collapse`）

**`Reveal`** —— mount 触发的一次性 fade-up + stagger。stagger 通过 inline `animation-delay` 注入（受 `STAGGER_CAP` 封顶）。

```tsx
// apps/desktop/renderer/src/components/Reveal.tsx
export function Reveal({ index = 0, children, className }: RevealProps): JSX.Element {
  // index 超过 STAGGER_CAP 不再累加延迟，避免长列表拖尾
  const delay = Math.min(index, STAGGER_CAP) * STAGGER_MS;
  return (
    <div className={`reveal ${className ?? ''}`} style={{ animationDelay: `${delay}ms` }}>
      {children}
    </div>
  );
}
```

**`Collapse`** —— 高度折叠。采用 **CSS Grid `grid-template-rows: 0fr → 1fr`** 技巧（无需 JS 测量像素，零 layout 抖动，对动态内容自适应），仅动 grid-rows + opacity：

```tsx
// apps/desktop/renderer/src/components/Collapse.tsx
export function Collapse({ open, children }: { open: boolean; children: React.ReactNode }) {
  return (
    <div
      className="collapse-track"
      style={{
        display: 'grid',
        gridTemplateRows: open ? '1fr' : '0fr',
        transition: `grid-template-rows var(--motion-slow) var(--ease-expo), opacity var(--motion-slow) var(--ease-expo)`,
        opacity: open ? 1 : 0,
      }}
    >
      <div style={{ overflow: 'hidden', minHeight: 0 }}>{children}</div>
    </div>
  );
}
```

> `grid-template-rows` 过渡在现代 Chromium（Electron 底座）全支持，比传统 `max-height` hack 更稳（无估高问题），比 JS measuring 更省（零 effect）。

### 6. 验收标准

1. **全量覆盖**：话术系统以下状态切换在 `balanced`/`full` 档均有可见动画——消息入场、marker pop、工具组展开/折叠、思考块展开、工具卡展开、running→done 切换、复制反馈、系统通知入场、流式光标、jump-to-bottom 入场。（逐条对照 §3.4 表）
2. **物理感**：主 easing 是 `expoOut`（非 `linear`/`ease`）；toggle/弹出用 `overshoot`；列表入场有 stagger。对齐 huashu-design 自检清单。
3. **门控生效**：`q-minimal` 档所有入场/展开动画退化为瞬切（与既有 `.ix-zone` 行为一致）；`prefers-reduced-motion: reduce` 下全部禁用动画。
4. **可配置**：改 `motion.ts` 的 `DURATION.normal` 或 `STAGGER_MS`，全局节奏随之变化，无需改组件代码。
5. **零新依赖**：`package.json` 不新增 runtime 依赖；打包体积无显著增长。
6. **无回归**：`typecheck` + `lint` + `build` 全绿；长会话滚动 ≥55fps（DevTools Performance 抽样）；明/暗主题切换动画不闪烁。
7. **易维护**：`styles.css` 每个 keyframe 有注释说明触发时机与意图；`motion.ts` 是单一改动源，组件不内联动画魔数。

## 附录 B · 与既有视觉系统的关系

- **F054（视觉系统刷新）**：本批次复用其全部语义 token（`--ok/--run/--info/--thinking/--warn/--danger`、`--accent*`），**不引入新颜色**——动画只动 transform/opacity/box-shadow，颜色走既有 token。

- **性能护栏（沿用 F060）**：对话滚动区（`.stream` / 消息列表）**绝不**挂 `backdrop-filter`；本批次动画只动 transform/opacity/grid-rows（GPU 合成或极轻 layout），不破坏滚动性能。
