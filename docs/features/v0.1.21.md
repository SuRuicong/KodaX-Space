# v0.1.21 - Patch Lane Release

> Release batch: patch-only
> Status: Released
> Source of truth: this file + [../FEATURE_LIST.md](../FEATURE_LIST.md) + [../../CHANGELOG.md](../../CHANGELOG.md)
> Release date: 2026-06-22
> Planning note: v0.1.21 is the first reserved patch lane; v0.1.22-v0.1.23 became follow-up patch lanes, v0.1.24 carries F105 customer timebox entitlement MVP, v0.1.25 carries F107/F108 SDK and composer catch-up, F103 starts at v0.1.26, and v0.1.27-v0.1.28 are reserved before the next feature pair.

---

## Intent

v0.1.21 does not assign new planned feature work. It keeps the v0.1.20 capability baseline intact and uses the first reserved patch lane to fix release-critical issues found while publishing and validating v0.1.20.

The release goal is:

**Keep the product baseline stable, make recovery/reporting more reliable, and make the release artifacts easier to consume.**

---

## Scope

- Workflow transcript restore notices are preserved when restored sessions are replayed.
- Workflow final reports render restored transcript content and empty-state recovery details more readably.
- Workflow manager details show restored final summaries directly before the full durable artifact is expanded.
- Workflow notice footers expose clearer affordances when restored or partial data is shown.
- Settings modal interaction tests are stabilized so CI fails on real regressions instead of modal timing noise.
- Packaged desktop builds include the keychain runtime files needed by `@napi-rs/keyring`, with macOS x64/arm64 release jobs split onto matching runner architectures.
- Windows release publishing includes zipped fallback artifacts for users who cannot use installer-style downloads.
- Root, desktop, IPC schema, UI kit, lockfile, capability contract, changelog, README, usage guide, capability ledger, and feature matrix are aligned to `0.1.21` / `space-v0.1.21`.

---

## Non-Goals

- No planned feature ID is assigned to v0.1.21.
- F103 does not start in this release; it remains planned for v0.1.26.
- F085-F094 do not move back into the patch lanes; they remain planned for v0.1.29/v0.1.30/v0.1.33/v0.1.34.
- KodaX SDK dependency remains on the v0.7.53 baseline used by v0.1.20.
- Full i18n foundation work remains under F076/F077/F078 rather than the v0.1.21 patch lane.
